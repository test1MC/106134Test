Devices "command delay" table

Last update 4/11/2016

Author: Gianluca Carraro

 

The following table contains the Command Delay value (in ms) set in blocks that manage devices (both CAREL and third parties)

Device Version Command Delay (ms) Note
c.pCOe_2 1.3.1 0 Fixed value in protocol template
CPY_2 1.0.1 4 Fixed value in protocol template
Energy meter mono 2.0.2 0 Fixed value in protocol template
Energy meter 3 1.0.2 4 Fixed value in protocol template
EVD EVO 2 1.4.0 4 Fixed value in protocol template
Humisonic 2 1.2.1 2 Fixed value in protocol template
KEC 1.0.0 0 Fixed value in protocol template
PSALB 1.0.0 0 Fixed value in protocol template
thT 1.0.0 0 Fixed value in protocol template
thTune 1.4.1 2 can be set by block input pin: default value is 2 and is set in the initial value box of the library POU template (for baudrate = 19200)
Wireless probe 1.0.0 0 Fixed value in protocol template
Serial probe 2 1.3.1 0 Fixed value in protocol template
Inverter powerplus 2.0.8 10 or 40 40 is set in DEV template (in the variable comment, it is suggested to set a value >= 40 ms).
It is present a Logic (in Demo project) that consider if the Power+ is a simulator or a real one: if it is a simulator, set the CmdDelay = 40, otherwise set it = 10
Kriwan In sviluppo 0 at the moment is 0; David F. is evaluating to set a different value (3/11/16)
Zhiel Abegg 2 2.0.0 2 set in DEV template; it is a block input pin
EBM pampst 2 1.2.0 2 Fixed value set in Library POU template (constant assignment); it is a block input pin. It is set also in the DEV template (no sense because constant assignment always wins)
