General operations

This function block controls a Copeland Scroll™ Digital Compressor.

Compressor capacity is modulated by controlling a solenoid valve. The solenoid valve can have two states:

  • valve closing (logic status TRUE): load phase when the compressor operates as a standard scroll and delivers maximum capacity;
  • valve open (logic status FALSE): unload phase when the compressor does not deliver any capacity.

Given the high number of transitions of the digital output needed to modulate capacity via solenoid control, SSR outputs must be used. Refer to the specifications for each c.pCO board.

Enabling the module

To enable the module, set En_CompScrollDigital = TRUE

Compressor capacity control

A Copeland Scroll™ Digital Compressor can modulate capacity from a minimum of 10% to a maximum of 100%.

To control capacity delivered by the compressor, manage the valve opening and closing times. The time restrictions are listed below:

18 sec

Cycle length: 12 sec 20 sec
Valve open: 2 sec Cycle length - valve closing
Valve closed: - 18 sec

 

Capacity control

CompScrollDigital manages up to three types of capacity control.

For all three control modes, if required capacity is under the minimum that can be delivered, the compressor will remain at minimum capacity until it shuts down.

For reduced capacity linked to the prevention procedure (see below), the delivered capacity cannot be below the minimum.

Optimised capacity control

The cycle length is set to 12 seconds with this control. Laboratory tests demonstrate that this is the time that provides maximum compressor efficiency.

The compressor cannot operate at under 16% of delivered capacity with this control

Capacity control with variable cycle length

Compared to optimised control, control with variable cycle length lets compressor capacity to fall below 16%.

Management is divided in two parts according to the required capacity:

  • Capacity ≤ 17% : cycle length is reversely proportional to required capacity and valve closing time is constant at 2 seconds.
  • capacity > 17% : cycle length is constant at 12 seconds and valve closing time is proportional to required capacity (as for optimised control)

Thus the compressor will have maximum output with demand higher than 17%, however this will diminish as it nears the minimum deliverable.

Capacity control with fixed cycle time

Cycle length control can be bypassed and manually set by pin. This increases control output which, with cycle lengths over 12 seconds, has a higher number of delivered capacity steps (to the detriment of compressor efficiency).

in some cases, a higher number of steps can ensure finer capacity control

 

Once the cycle length has been set (minimum 12 seconds), the valve closing time is directly proportional to required compressor capacity.

As can be seen, when required capacity is 100%, the valve closing time is equal to cycle length thus the compressor delivers maximum capacity.

Start up

Copeland Scroll™ Digital Compressor has a start up procedure divided into three different phases:

  1. on compressor start request, the bypass valve starts to minimally balance compressor pressure and ensure a reduced initial load;
  2. after 4 seconds the compressor starts and the valve starts the working cycle, providing 50% load to the compressor for three minutes;
  3. next, the valve closes and the compressor delivers maximum capacity for one minute.

During the start up phase, CompReq_SCRDIG is ignored. At the end of the start up procedure, capacity delivered by the compressor starts to follow the request (CompReq_SCRDIG). If the request is cancelled during the start-up phase, the algorithm turns off the compressor, however observing the safety times.

EVD evolution management filter

For good circuit operation that includes a digital scroll compressor and an electronic expansion valve, the valve driver must be coordinated with the capacity delivered by the compressor.

For this reason, CompScrollDigital includes a control that calculates the value of a filter to be sent to the EVD.

Manual Mode

Manual mode is performed outside this function block because the safety time management and power request calculation are realized respectively by CntDwnSafT, DevAvblStatus and EnvPwr.
