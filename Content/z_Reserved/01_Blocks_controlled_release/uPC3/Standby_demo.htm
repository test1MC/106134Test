Project description

This demo shows how to manage the Stand-by function of the uPC3 either from the stand-by DIN or from the PLDpro display.

 

For more information about standby mode, see enrico.perazzolo@carel.com

 

The demo uses the following C Function Blocks:

C Function Block name Parameter Use for
SYS_READ SYS_PARAM_GET_JST_INPUT Read the input from DIN
SYS_WRITE SYS_PARAM_SYSTEM_POWERDOWN 181818 to switch off from PDLPro
191919 to swicth off from DI

 

Main settings
Standby_CommandReceived This variable must be managed by your application, if TRUE a stand-by command has been received either from the dedicated DIN or from mask, use the status of this variable to secure the machine (i.e. disable all devices)
Standby_ProcedureCompleted This variable must be managed by your application, set this variable to TRUE once the machine is secured, when TRUE the uPC3 will go in stand-by
Standby_Mng.Standby_UI This variable is used to request the stand-by procedure from the UI.

 

If the uPC3 is put in stand-by from the stand-by DIN, the uPC3 can be waked up only from the DIN, the same behavior applies to the stand-by from mask.

The stand-by from mask is available only with PLDPro with codes: PLD**GFP00.
