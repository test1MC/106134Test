General operations
Minimum device on time

This feature guarantees the minimum running status of the device (MinOnT), once it has been switched on.
The timer starts when DevStatus = TRUE.

Minimum device off time

This feature garantees the minimum off status of the device (MinOffT), once it has been switched off.
The timer starts when DevStatus = FALSE.

Minimum time between two starts of the same device

This feature garantees the minimum time between two starts of the same device (MinOnOnSameT).
See diagram above.

CntDwnTSaf will be the higher between MinOnOnSameT and MinOffT when DevStatus = FALSE

DevAlrm

The DevAlrm input immediately stops the device (DevOn = FALSE), also bypassing the MinOnT.
See the diagram in chapter "Minimum device on time".

FrcdOnByT

The output FrcdOnByT is TRUE if there is no request for activation of the device, nevertheless it remains on (DevReq = FALSE, DevOn = TRUE) for MinOnT.

FrcdOffByT

The output FrcdOffByT is TRUE if there is a request of activation, yet it still stays off (DevReq = TRUE, DevOn = FALSE).
This condition occurs when the device is off due to one of the following functions:

  • minimum off time (MinOffT)
  • minimum time between starts of the same device (MinOnOnSameT)
