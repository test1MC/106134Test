Creating a new solution from Project Template

First case: creating a new solution in c.strategy
  1. From c.strategy, in the File menu, select New, Project
  2. Select Carel project Template, enter the new project properties (Name, Path) and click OK to confirm

Second case: import the Project Template into another project
c.strategy
  1. In the new project, add Dependencies “CAREL_BASIC_LIB”, “VersionChk_2”, “Blackout_3”, "CheckRetainMem" and "TZ_MNG"LIB" (right click on “ProjectName”, add --> add dependency)
  2. Export POUs from c.strategy and import them into the new project
  3. From “Global variables” in Project Template
    1. click“Export”
    2. in the new project, click“Import” and then “Import Flags” giving the same file (always from “Global variables”)
  4. Export the alarm table and import it into the new project
  5. Build the solution
c.mask
  1. In the new project, delete the empty worksheet
  2. Export worksheets from the Project Template and import them into the new project
  3. From the Project Template
    1. open code.st
    2. select all (ctrl+A)
    3. copy (ctrl+C)
    4. paste into the new code.st project (ctrl+V), overwriting all the code
  4. In the new project, open a worksheet, click in the white field and, in the “KeyBinding Editor” set
    1. for ALARM key --> “CheckActiveAlarm”
    2. for ESC key --> “GlobalEsc”
    3. for PRG key --> “GotoMainMenu”
  5. Export translation Excel file (right click on Layout (PGD1) --> Export --> Translations) and, before importing it into the new project
    1. In the new project, right click on Layout (PGD1) --> Languages
    2. Add, in order: English [EN], Italian [IT], German [DE], French [FR], Spanish [ES]
    3. Save and import the Excel file into the new project (right click on "Layout (PGD1)" --> Import --> Translations)
  6. In the “Properties” of a worksheet (any one), at item “BeginFromMask” select “CarelLogo”
  7. Save, close both c.strategy and c.mask, reopen and rebuild all: no error should appear
c.design
  1. Export c.design configuration (on “Select configuration” page), open c.design for the new project, click cancel (for the new configuration) and then import.
Third case: start from the latest version of the Project Template

This can be useful if needing to start a new project from the Project Template, however c.suite does not have the latest version of the template.
Then:

  1. Go to the KSA website at the following link and download the latest version of the Project Template
  2. Open the project, change the solution and project name
Update the template or add your own template

To update the template with the last version or to add your own template to c.suite follow the steps below:

If you have to update the Project Template download the last version from KSA at the following link

  1. Open the template you want to add
  2. In "File" --> press "Export Template"
  3. In "File" --> press "Import Template" and select the file exported before
  4. Reboot c.suite

If you need to remove the template, remove the files at the following paths:

  • C:\Users\("user name"\AppData\Roaming\CAREL\Acp\("c.suite version")\ProjectTemplatesCache

  • C:\Users\("user name")\Documents\Carel c.strategy\Templates\ProjectTemplates
