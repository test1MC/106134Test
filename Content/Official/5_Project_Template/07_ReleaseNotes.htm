Changed
• Changed application cycle time from 0 ms to a fixed cycle time of 500 ms: a fixed cycle time allows other OS tasks (like Protocols, mask, logger and so on) to have more time slot for their execution. (See "Cycle Timing" in c.strategy Glossary)

500 ms is a typical application cycle time that doesn't compromise the correct functioning of the application. (CAREL_Ref Ticket #296)
To change or remove the fixed cycle time, set the two Resource properties "Cycle time" (for the time) and "Trigger Cycles" (for the enable).
