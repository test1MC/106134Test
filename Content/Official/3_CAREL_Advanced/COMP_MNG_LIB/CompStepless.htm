General operations

The compressor starts when the CompReq_STPLS input exceeds the minimum control capacity percentage (CfgCompStepless_STPLS.MinPwr). Using the information in the CfgCompStepless_STPLS structure, working phase and feedback from valve run (VlvPulse_STPLS), the function block provides an estimate of the percentage of capacity delivered by the compressor (PwrRun_STPLS).

Compressor start depends on compressor status feedback and only occurs if one of the following conditions is not included:

DevAvlb_STPLS = 11 Compressor in OFF due to safety time

DevAvlb_STPLS = 6 Compressor forced OFF due to non-serious alarm (normal shutdown phase)

DevAvlb_STPLS = 10 Compressor forced OFF due to serious alarm (immediate shutdown)

The compressor does not shut down if timer feedback includes the minimum ON time in progress conditions (DevAvlb_STPLS = 12).

 

Start/stop control

Before starting control, the compressor runs a start-up sequence (generally recommended by manufacturers). This sequence is made up of two phases: in the first, the compressor is brought to the minimum deliverable capacity (CfgCompStepless_STPLS.StartStopPwr ) for the time set in (CfgCompStepless_STPLS.StartUpT1). The purpose of the second is to bring the compressor to minimum control capacity (CfgCompStepless_STPLS.MinPwr) for the time set in (CfgCompStepless_STPLS.StartUpT2).

A shutdown request (CompReq_STPLS = 0.0) causes similar behaviour: in the first phase, the compressor is brought to minimum control capacity (CfgCompStepless_STPLS.MinPwr) for the time set in (CfgCompStepless_STPLS.ShOffT1). Next, the compressor is brought to minimum deliverable capacity (CfgCompStepless_STPLS.StartStopPwr) for the time set in (CfgCompStepless_STPLS.ShOffT2). This sequence is also generally recommended by manufacturers.

However, the shutdown phase can be bypassed in two ways:

  • DevAvlb_STPLS = 10: compressor forced OFF due to serious alarm
  • ShOffRes_STPLS = 1: the shutdown phase is no longer necessary and the compressor can be turned off

The individual phases can be skipped by setting the corresponding time to 0s..

 

Control

CompStepless implements a calculation logic for the capacity delivered by the compressor based on the time it takes to reach maximum capacity (CfgCompStepless_STPLS.DT_MaxPwr) and the time it takes to reach minimum capacity (CfgCompStepless_STPLS.DT_MinPwr): these parameters indicate the time required for the slider to move from the minimum to the maximum capacity position (100%) and from the maximum (100%) to the minimum, always keeping the load/unload valve on.

Since the valve is controlled at known pulses (T_Incr_STPLS and T_Decr_STPLS inputs), the algorithm calculates the screw compressor capacity variation percentage at each valve start pulse, based on the compressor phase (Increase or Decrease).

CompStepless checks valve behaviour through the VlvPulse_STPLS input: before entering the increase or decrease phase, it checks whether valve start can cause over capacity delivery or under elongation and, in this case, the compressor goes into standby.

when maximum/minimum capacity is reached, CompStepless forces the increase/decrease phases (alternated with standby) to make sure the slider remains in the maximum/minimum capacity position.
