General operations

The compressor starts when the CompReq_STP input exceeds the minimum start threshold. According to the number of load steps, an increasing output capacity power percentage is provided with variable power steps according to the input list settings.

The compressor only starts occurs if the compressor status feedback does not include one of the following conditions:

DevAvlb_STP = 11 Compressor in OFF due to safety time

DevAvlb_STP = 6 Compressor forced OFF due to non-serious alarm (normal shutdown)

DevAvlb_STP = 10 Compressor forced OFF due to serious alarm (immediate shutdown)

The compressor does not shut down if timer feedback includes the minimum ON time in progress conditions, thus DevAvlb_STP = 12.

 

Start/stop control

Before control starts, the compressor must complete a start-up time. During this phase, the CompStep function block does not proceed with control as per the CompReq_STP request but inhibits compressor start-up for the corresponding time (CfgCompStep_STP.StartUpT). The capacity delivered (PwrRun_STP) corresponds to start-up capacity (CfgCompStep_STP.StartStopPwr).

This procedure is also followed for compressor shutdown. When a shutdown request is received (CompReq_STP = 0), the compressor does not immediately shut down but remains in the shutdown phase for the corresponding time (CfgCompStep_STP.ShOffT). The capacity delivered (PwrRun_STP) corresponds to the start-up capacity (CfgCompStep_STP.StartStopPwr).

However, the shutdown phase can be bypassed in two ways:

  • DevAvlb_STP = 10: compressor forced OFF due to serious alarm
  • ShOffRes_STP = TRUE: the shutdown phase is no longer necessary and the compressor can be turned off

 

Load step start

Load step start is subject to set times; the various starts are divided by a variable time according to the compressor phase.

The function block only signals the compressor load step start request through pin PhRun but the physical digital output activation is controlled by logic external to the function block.
