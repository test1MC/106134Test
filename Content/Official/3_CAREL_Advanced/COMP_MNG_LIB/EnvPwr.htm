EnvPwr

Manages compressor power request, fan set point and electronic expansion valve MOP values to be sent to the EVD function block to maintain and return the working point within envelope limits; it also manages the cooling capacity value to be sent to the EVD to control the electronic expansion valve.

General operations

EnvPwr contains an advanced algorithm, developed by CAREL and password protected.

EnvPwr runs the following functions based on the working zone defined by the input pin ZoneEnv coming from EnvAlrm FB:

  1. checks compressor variation speed according to distance from edge
  2. corrects power request to return within envelope or discharge temperature operating limits
  3. provides a set point limit for condenser fan modulating control in chiller and heat pump modes
  4. corrects the fan control limit when needed to prevent the working point from exiting the envelope
  5. checks electronic expansion valve start and opening sending the MOP and cooling capacity values to the EVDEVO function block

In the start phase, EnvPwr checks the pressure differential value and keeps the electronic expansion valve closed until the minimum pressure differential between discharge and suction is reached to guarantee lubrication; the valve is activated in any case if, during the start phase, the minimum evaporation pressure threshold is reached before the nominal pressure difference is reached.

Normal power limitation management in zone 9 is disabled during the start phase and for the entire period set by pin StartupDT. This allows working conditions to be quickly reached (see generic envelope illustration).

Inhibition limit management for MOP protection

The MOP protection limit is entered as a saturation temperature value to easily compare it with the compressor manufacturers' technical specifications. Protection triggers to prevent too high evaporation temperatures from causing an excessive work load for the compressor with consequent motor overload and possible thermal overload protection triggering. Inhibiting this limit and changing the MOP value in real-time, the algorithm can correctly manage the compressor operating zones. Pin MOPThrshFastChg must be linked to EVDEVO.

 
High discharge temperature management

The compressor working zone is also limited by high discharge temperature which typically occurs when refrigerant flow is poor due to low suction pressure and high discharge pressure (meaning when the compressor working zone is near the maximum compression ratio limit, near zone 2).

Similar to envelope limit control, temperature is controlled at discharge by a specific algorithm that initially slows acceleration to inhibit it when set by control. If the temperature continues to climb, the control algorithm gradually and slowly manages speed reduction to imitate the compressor's heat inertia behaviour.

Preventive actions

The actions run by the function block control algorithm are listed below to:

1. Check that the working point is inside the envelope;

2. Manage return within the envelope if the limits are exceeded, based on the zone specific to the indicated limit.

Condenser fan control set point management in chiller mode

The fan control set point can be obtained by adding an offset to the CondRegLimSetTemp output pin value, which corresponds to the minimum envelope condensation limit (low limit in the illustration below).

To optimise envelope control and compressor efficiency, this minimum value is increased, adding a correction, when the compressor working point is near the envelope zone 6 or zone 7 limit (see following diagram), within 5°C of the vertical projection on the limit. Within this zone, the set point increases at a rate of 1.2°C/min; if the compressor working point exits the envelope in zone 6 or 7, the offset increases at a rate of 3.6 °C/min; as soon as the working point returns above the specified increase zone, any offset decreases at a rate of 1.2°C/min until it cancels itself out.

 

Evaporator fan control set point management in heat pump mode

The fan control set point can be obtained by subtracting an offset from the EvapRegLimSetTemp output pin value, which corresponds to the maximum envelope evaporation limit (High limit in the illustration below).

To optimise envelope control and compressor efficiency, this maximum value is reduced subtracting a correction, when the compressor working point is near the envelope zone 6 or zone 7 limit (see following diagram), within 5°C of the horizontal projection on the limit. Within this zone, the set point decreases at a rate of 1.2°C/min; if the compressor working point exits the envelope in zone 6 or 7, the offset increases at a rate of 3.6 °C/min; as soon as the working point returns above the specified increase zone, any offset increases at a rate of 1.2°C/min until it cancels itself out.

 

 

Special functions: compressor positioning at the requested capacity

To allow reverse cycle or other operations to start, required capacity at block input can be forced using pin GotoPwrReq: switching pin value to TRUE inhibits block capacity control and the value at input is copied to the output, with the only limit or meeting any maximum capacity limit (signalled by pin MaxCapZone).

 

RfrgTyp pin

This pin is used to convert the P-T values acquired by the probe in high resolution; the value refers to the list accessible in the PressToTemp_REAL help, consistent with the envelope data in the input list.

 

EnvP_X and EnvP_Y

The envelope vertex coordinates calculated and used by the function block are read in the EnvP_X and EnvP_Y input lists from position 0 to 7 as x and y values corresponding to the points indicated in the envelope examples shown in the following illustrations. The envelope on the right shows how values are assigned to the coinciding points (P1=P2 and P6=P7) due to lack of the relevant segment.
