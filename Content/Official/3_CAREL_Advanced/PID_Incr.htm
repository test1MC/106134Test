General operations

This block is used to calculate the capacity request for a device in order to reach the control set point.

The algorithm uses Proportional, Integral and Derivative terms (all of them, or only some) to achieve best performance.

The integral or derivative part can be inhibited by setting the relevant action time to zero (Ti_PID_Incr=0 to avoid integral, Td_PID_Incr=0 to avoid derivative).

The formulas used are those relating to incremental PID, where the output is the result of the sum of the three calculated deltas between two different samples of the input variable.

This algorithm doesn't use the control band parameter as an input, this means that the Function Block uses the proportional gain (Kp).

 

Sampling time

The Period_PID_Incr input parameter is the sampling time.

At each settable period, the RegVar_PID_Incr input parameter is sampled and the delta is calculated between two samples.

 

Dead band

The function intends to keep the output value stable as long as the input remains within the DeadBand. DeadBand is the range between [Setpoint+(DeadBand/2)] and [Setpoint-(DeadBand/2)]. The function is usually disabled and can be enabled by setting the DeadBand input parameter to a value greater than 0.

 

Cool/Heat mode vs Direct/Reverse mode

Cooling/Heating mode refers to the calculation of the delta between two different samples of the sampled input variable. It depends on whether the meaurement is above the set point (cooling) or below the set point (heating).

Direct/Reverse mode acts on the PID output.

Direct action means that the controller output rises if the measurement increases. Indirect (reverse) action means that the controller output drops when the measurement rises.

Output limitations

With LowLim_PID_Incr and HighLim_PID_Incr is possible to limit the output (Req_PID_Incr) to a range of values.

The default values of these inputs must be set to 0.0 and 100.0 if no limitations are required.

LowLim_PID_Incr must and HighLim_PID_Incr must be greater than or equal to 0.0

 

Manual mode

When En_ManMode_PID_Incr is TRUE, the algorithm is bypassed and the Output takes exactly the same value as ManModeVal_PID_Incr in accordance with the maximum and minimum limits.

When manual mode is enabled, the algorithm constantly keeps the samples of the input variable updated. It allows the PID to calculate the next delta starting from the previous value in manual mode.

If En_PID_Incr is FALSE, the Req_PID_Incr is 0.0 also if in manual mode.

 

Enable and reset

If En_PID_Incr = FALSE, the Req_PID_Incr is always 0.0 and the entire algorithm is bypassed. All internal errors and temporary variables are reset.

This input can be used for resetting the PID.

To disable the PID, it is not sufficient to simply avoid calling it (in ST language), because in this way it is not reset and all errors continue to increase, only En_PID_Incr can disable and/or reset the function.

 

Using the algorithm with only the proportional part

It is possible to set the integral and derivative parts to zero and use only the proportional part.

In this case we need to take into account the PID formula in incremental form when considering only the proportional part:

Delta proportional term := Kp * (Current measurement - Previous measurement )

Output = Current Output + Delta proportional term

Consider the following example:

SetP_PID_Incr = 20.0

Kp_PID_Incr = 5.0

Low output limit = 0.0

High output limit = 100.0

  1. Set RegVar_PID_Incr = 25.0
    Current output = 0.0
    Delta proportional term = 100.0
    Req_PID_Incr = 0.0 + 100.0 = 100.0
  2. Set RegVar_PID_Incr = 28.0
    Current output = 100.0
    Delta proportional term = 60.0
    Req_PID_Incr = 100.0 + 60.0 = 100.0
  3. Set RegVar_PID_Incr = 24.0
    Current output = 100.0
    Delta proportional term = -80.0
    Req_PID_Incr = 100.0 + (-80.0) = 20.0
