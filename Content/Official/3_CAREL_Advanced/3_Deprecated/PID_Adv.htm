General operations

This block is used to calculate the capacity request for a device in order to reach the control set point. The algorithm uses Proportional, Integral and Derivative terms (all of them, or only some) to achieve the best performance.

PID_Adv is designed to be compliant with IEC 61131 - 3 standard rules and algorithms.

This function blocks implements PID control with ancillary functions. Specifically, PID_Adv can:

  • enable the Anti-Bump function;
  • check output saturation limits.

The integral or derivative part can be inhibited by setting the relevant action time to zero (Ti=0 to avoid integral, Td=0 to avoid derivative).

Proportional action

As shown in the graph below, a stationary steady state error always occurs with proportional control only, which tends to reduce with increasing gain used. In the graph, the feedback to a P control system unit step, as Kp changes (example from literature):

The proportional gain is sent directly to the pin. The proportional contribution, as the name indicates, is directly proportional to the extent of the error

Output = Kp [ SetP - Input]

 

Integral action

The integral action contribution increases and the Ti coefficient decreases. The integral action is proportionate to the integral at the time of error. The graph shows that, using the integral component, the steady state error disappears. In fact, this term integrates the time error and thus a constant error increases the control action until the error is fully cancelled out.

In the graph, the feedback to a PI control system unit step as Ti changes (example from literature)

Derivative action

The derivative action represents the signal movement speed. This action is to compensate for the deviation due to its change speed, without waiting for the error to become significant (proportional action) or that it persists for a certain amount of time (integral action).

The purpose of the derivative action is to introduce an action that is proportional as the error changes, thus providing a correction that "anticipates" the error trend in time.

systems do no always need this action, they often have simple P or PI controllers. In fact, derivative action controllers have the problem of amplifying signals with harmonic contribution at high frequencies such as, for example, noise overlapping the working signal.

Control band

PID_Adv uses Kp (proportional gain) for setting the proportional action as in the IEC 61131-3 standard .

Kphas no unit of measure.

The set point is in the middle of the control band; for instance, if the setpoint is 20.0°C and the control band is 4.0°C, the Output will be equal to HighLim when temperature is 22.0 °C and will be equal to LowLim when temperature is 18.0 °C.

The formula to calculate Kp is Kp = 200.0 / CtrlBand. Example: if control band is 4.0°C the corresponding Kp will be 50.0.

Cycle time

Cycle is used to reduce wastedpCO processing power by executing the algorithm only one time every Cycle milliseconds. Values lower than 500 ms can cause PID behaviour to be highly dependent on application cycle loop time.

Antibump (bumpless transfer)

This function acts when PID working mode passes from manual to automatic mode or when a control parameteris changed. Normally, in one of these cases, the Output will have a bump. Antibump avoids this behavior by modifying the integral part to ensure that the Output initially remains the same and reaches the right value slowly.

The function is normally disabled and can be enabled using input pin En_AntiBump.

Deadband

The function intends to keep the output value stable as long as the input remains within the DeadBand. DeadBand is the range between [Setpoint+(DeadBand/2)] and [Setpoint-(DeadBand/2)]. The function is usually disabled and can be enabled by setting the DeadBand input pin to 1.

When the Input is in deadband, the integral error is frozen.

Example:

Orange band is the deadband (2.0°C).

When the temperature (green line) enters the deadband, the output (white line) remains constant.

Instead, when temperature exits from the band, the PID continues with normal control.

Output limitations

With LimLow and LimHigh is possible to limit the Output to a range of values. The default values of these inputs must be set to -100.0 and 100.0 if no limitations are required.

The function block uses these values for the antiwind-up function, so putting a limit on Output outside the function block instead of using these inputs can be unsafe. In this case, it cannot be ensured that wind-up phenomena will be avoided.

The limits act only on the Output, not on the internal calculation. This means that the real output values are still based on the control parameters even if the limits are different from -100.0 and 100.0.

Example:

Orange band is the limitation of the Output with LowLim = 30.0 and HighLim = 80.0. Output never goes out of this band (except when En_PID is false, in that case Output is 0.0).

Antiwind-up

PID_Adv checks the relationship between the proportional and integral error in run time. Without Anti-Windup, when the saturated PID output reaches the maximum limit set by the pin, the integral action continues to integrate the error. When the output reaches the set point, it waits until the integral term discharges before it starts to lower towards the set point, thus generating over-elongation.

The Anti-Windup blocks the ntegral error calculation and forces the difference between the maximum limit and proportional term. If the latter is higher than the maximum limit, the integral is brought to zero. If Ti is zero, Anti-Windup is automatically disabled.

Manual mode

When ManMode is TRUE, the algorithm is bypassed and the Output takes exactly the same value as ManOut in accordance with the maximum and minimum limits. When the PID works in manual mode, the integral error is reset as in the IEC 61131-3 specifications. If En_PID is false, the Output is 0.0 also if in manual mode.

It is strongly recommend to enable the antibump function when using manual mode.

Enable and reset

If En_PID = FALSE, the Output is always 0.0 and the entire algorithm is bypassed to avoid wasting pCO processing power. All internal errors and temporary variables are reset.

This input can be used to reset the PID.

To disable the PID, it is not sufficient to avoid calling it (in ST language), because this way it is not reset and all errors continue to increase; only En_PID can be used to disable or reset the function.
