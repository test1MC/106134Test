General operations

The FB blocks inside PriorityMNG_LIB, manage the case of a parameter that could be changed simultaneously by mask/algorithm or/and parameters from serial.

It is necessary to create a function block that manages such concurrency.

This management has to:

 

  Case Action Result
1 When the input value (used in the mask/algorithm) changes It must be copied to the shared variable protocol The mask/algorithm has priority
2 When the shared variable protocol changes It must be copied to the variable used in the mask/algorithm The protocol has the priority
3 If both variables change during the same program cycle The value of the variable used in mask/algorithm has priority and must be copied to the shared variable protocol The mask/algorithm has priority

 

PriorityMNG_LIB contains one version of the FB for each datatype:

  • PriorityMNG_Real
  • PriorityMNG_Int
  • PriorityMNG_Bool
  • PriorityMNG (used for DINT)

 

Each Priority manager FB can be replaced by a custom version of the Block which could include more than one variable or structured variables. Many Applications use this function as imported ST code and applied to the variables.

The same Input and Output has to be flagged as user interface variable (See picture 1).

Modbus variable has to be flagged as protocol (See picture 1).

      	IF VariableIn=SerialIn THEN

      	IF VariableIn<>SerialIn AND VariableIn<>Variable THEN
	    (* Case 1: the input changes        *)
	    Variable:=VariableIn;
	    Serial:=VariableIn;
    	    UpdtDn:=TRUE;
    	    (* Case 3: If both are changed, the input variable
	    has priority and must be copied into protocol.
	    "UpdtDn:=TRUE;" ==> jump the rest of the code         *)
      	END_IF;
      	IF VariableIn<>SerialIn AND SerialIn<>Serial AND NOT UpdtDn THEN
	    (* Case 2: the protocol changes     *)
	    Variable:=SerialIn;
	    Serial:=SerialIn;
      	END_IF;
