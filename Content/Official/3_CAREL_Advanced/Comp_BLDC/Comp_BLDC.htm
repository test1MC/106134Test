General operations

Comp_BLDC manages the capacity request to keep the compressor in operating conditions within the pressure and temperature limits set by the compressor manufacturer.

Comp_BLDC internally includes the "InverterPowerPlus" function block.

The image below shows the flow diagram of the operating principle of Comp_BLDC.

Comp_BLDC works with the EVD EVO thermostat valve function block.

 

Comp_BLDC only works when combined with the CAREL Power+ inverter: if there isn't communication with the inverter, the compressor cannot be started. In this situation, the AlrmCompOffflag is activated and the "Incompatible inverter model" alarm appears after 30 seconds.

Comp_BLDC is compliant with the qualification tests conducted by the specified compressor manufacturers.

Limiting actions, aimed at keeping the compressor within the operating range specified by the manufacturer, are divided into:

  1. reducing acceleration or deceleration when the compressor working point nears the limits, even blocking speed when the working point is at the limit;

  2. reducing speed, when the working point is beyond the high condensing pressure or high compression ratio limit, to stay within the allowed working range.

  3. increasing speed when the working point is under the minimum condensing pressure limits or minimum DeltaP or minimum compression ratio or over the maximum suction pressure (envelope_Zone = 5,6,7 or 8), to stay within the allowed working range (it can be enabled from parameter as an option).

The function block also provides some functions run with the EVD EVO thermostat valve driver to check the maximum current limit, the low compression ratio limit and the low delta P limit and the maximum evaporation pressure limit. The low compression ratio limit and low delta P limit can be controlled (alternatively, with higher global efficiency) by evaporator or condenser fan speed control, if installed, using the specific output pins.

The compressor type is automatically selected by setting the Inverter Motor Type parameter on the user interface. The Comp_BLDC function block automatically sets the type of refrigerant to be used (Refrigerant output parameters) based on the selected compressor type.

 

Enabling the Function Block

To enable the function block, set En_Comp_BLDC = TRUE.

 

Out of envelope alarm

The EnvAlarm is generated when the compressor working point remains out of the envelope zone for too long. The zone number is indicated in the alarm.

The alarm turns off the compressor. To restart the compressor, reset the alarm using the AlrmEnvRes input parameter.

During maintenance (the minimum compressor operating time), out of envelope alarms are inhibited but corrective speed control remains on in the event the upper envelope limit is neared or surpassed (area 2, 3 and 4 limits).

Alarm management

The Comp_BLDC function block manages out of envelope alarms in two different ways: manual or semi-automatic. Semi-automatic reset attempts to automatically reset envelope alarms. Manual reset is required when the maximum number of automatic attempts has been reached.

The default reset mode for these alarms is manual.

The "manual" or "semi-automatic" reset configuration is contained in the structure BLDC_AlarmMng_struct.

See also BLDC_AlarmMng_Struct

Each new alarm acquired by the Comp_BLDC function block starts a time frame set in TimeThrs. Envelope alarms are automatically reset in this time frame.

Reset attempts are incrementally counted on each new acquired alarm reset. The count is only decreased at the end of the time frame, started by the alarm rising front (See graph).

The Comp_BLDC function block signals envelope alarm status and the automatic reset attempt in progress through variable Info_ResetRetryWaiting. Variable Info_ResetRetryExpired indicates the need to manually reset the alarm, this condition occurs when:

• the maximum number of attempts available in the TimeThrs time frame are timed out.

• a single manual reset alarm is triggered

The manual reset command resets all automatic alarm reset attempt timers and counters.

The numbers used in the graph above have the following meanings:

  1. Out of envelope alarms.
  2. Alarm generated by the Comp_BLDC function block.
  3. Automatic reset started featuring a 30 s pause time before reset is sent.
  4. The time frame set for the first alarm event starts.
  5. Automatic reset attempt number count increases in the time frame set for the first alarm event. Number of attempts: 1.
  6. The time frame set for the second alarm event starts.
  7. Automatic reset attempt number count increases in the time frame set for the first alarm event. Number of attempts: 2.
  8. Automatic reset attempt number count decreases in the time frame set for the first alarm event. Number of attempts: 1.
  9. Automatic reset attempt number count decreases in the time frame set for the second alarm event. Number of attempts: 0.

Refer to DEMO_Comp_BLDC for semi-automatic alarm reset settings.

 

 

Discharge superheat management - Siam compressors

Similar to suction superheat control, a specific algorithm controls discharge superheat. The valve opens and closes in order to keep the required set point. The control algorithm works according to a set point and differential. It triggers when the input value increases and turns off when it decreases. Set variable EvdReg_SubTypeRegMode =2 inside structure CfgEnvCtrl_BLDC to enable this function.

The algorithm is set by controller specifications.

 

Inhibition limit management for MOP protection.

The MOP protection limit is entered as a saturation temperature value to easily compare it with the compressor manufacturers' technical specifications. The protection is triggered to prevent too high evaporation temperatures from causing an excessive work load for the compressor, with consequent motor overload and possible thermal overload protection triggering. Inhibiting this limit and changing the MOP value in real time, a special algorithm is able to correctly manage compressor work areas. Inside the structure BLDCtoEVD, the MOPThrshFastChange and MOPInhThrsh variables must be connected directly to the MOP_ThrshFastChance and MOP_InhibThrshvariable inside the structure of the EVD EVO function block.  

 

High discharge temperature management

The compressor working zone is also limited by high discharge temperature which typically occurs when refrigerant flow is poor due to low suction pressure and high discharge pressure (meaning when the compressor working zone is near the maximum compression ratio limit, near zone 2).

Similar to envelope limit control, temperature is controlled at discharge by a specific algorithm that initially slows acceleration to inhibit it when set in the control; if temperature continues to climb, the control algorithm gradually and slowly manages speed reduction to imitate the compressor's heat inertia behaviour.

Figure 3

If discharge temperature exceeds the maximum set limit, a discharge high pressure alarm is generated, turning off the compressor. The operator must reset it.

 

Compressor start/stop management

Start: when the CompPower0_100 input signal exceeds the minimum speed, a compressor request is generated (internal parameter CompOn initiates the start procedure).

Stop: when the CompPower0_100 input signal drops under 2%, the shutdown procedure starts. Even resetting the variable connected to the Comp_On input pin starts the shutdown procedure (with maximum deceleration ramp).

Emergency shutdown or shutdown due to an alarm

Triggering the CompAlrmShutOff input immediately shuts down the compressor without running the deceleration ramp.

Optimal lubrication

Comp_BLDC provides the best start possible, working with controlled devices, to guarantee that the operating conditions required to guarantee optimal lubrication are met as quickly as possible.

To do this, when started, the thermostatic valve is kept closed until the condensing-evaporation pressure difference exceeds the "Minimum Delta P". It is possible to change this threshold using the parameter StartUP_DeltaPPercStartEVD inside the structure CfgEnvCtrl_BLDC; the inverter is controlled to quickly bring the compressor to the set start speed and this value is maintained for the entire duration of MiscMng_BLDC.Time_MinOnT, as indicated in the example profile in Figure 4.

There is a control that inhibits the above-described thermostatic valve opening delay if suction pressure is at or under the minimum suction limit (zone 9). The purpose of the function is to improve behaviour at start up when the amount of refrigerant in the evaporator is quite high and does not allow discharge pressure to "rise"; immediate thermostatic valve opening provides refrigerant to the evaporator, overcoming the critical situation which would otherwise generate a low pressure alarm in the start-up phase.

During the phase of start-up and warming (which corresponds to the minimum operating time of the compressor), out of envelope alarms are inhibited but corrective speed control remains on in the event the upper envelope limit is neared or surpassed (zone 2, 3 and 4 limits).

Figure 4 (i.e.: Siam compressor)

The low pressure difference alarm AlrmLowDeltaP is generated when the difference between the condensing and evaporation pressures (Delta P) persists for a period of time under the limit value (specified by the compressor manufacturer) and thus optimal lubrication is not guaranteed. The alarm turns off the compressor (which needs to be reset by the operator). The alarm is inhibited when input parameter SpecifParam = TRUE) (typically during defrost); the out of envelope alarm remains on, which also includes low DeltaP, with the delay specified by input parameter CfgEnvCtrl_BLDC.CstmEnv_EnvOutMaxT.

Compressor start is inhibited if the difference between discharge and suction pressure is greater than the limit set for each type of compressor: the limit is due to start torque management which is limited by the maximum current deliverable by the inverter. In this case, a thermostatic valve pre-start-up procedure is initiated for a maximum time: when the pressure difference returns under the maximum limit, the start-up procedure is initiated. If the system has a pressure equalisation solenoid valve, a specific parameter can be set to activate this solenoid valve (output parameter CmdEqVlv) instead of thermostatic valve pre-start.

Failed compressor start management

A specific algorithm checks whether the compressor is actually started and, if this is not the case, signals failed compressor start through an alarm. Restart is automatically attempted several times until the set maximum attempts (default 5) are reached, blocking automatic restart in the event of failure.

Upon the fifth failed attempt (Default value), the alarm must be reset by the operator.

Failed start is caused by the following conditions that occurred in the first ten seconds from when the start signal is sent to the inverter module:

• failed increase in the condensing-suction pressure difference;

• "over current" or "no output phase" or "motor block" error code received by the inverter module.

In general, the five attempts (default value) are exceeded due to an incorrect setting (parameters with incorrect values) or wiring errors (incorrect compressor phase sequence).

 

Special functions: compressor slowing to minimum speed

To allow cycle reversal or other operations preferably run in less stressful conditions for the compressor, use input parameter GoToMinSpeed: switching the variable value to TRUE to slow speed with maximum deceleration to the minimum specified MinSpeedRpsGoTo. This minimum speed value is kept for as long as the input parameter value is TRUE; when the value is reset, speed returns to the set value with the acceleration set by control or by input parameter Speed_MaxGoUpSpeedRate (if enabled by pin SpecifParam=TRUE).

 

Special functions: Custom envelope limits

To meet the manufacturer's specific needs, the envelope range on maximum discharge and minimum suction pressures can be reduced. In this way the compressor is controlled (speed reduced due to having exceeded the limit) before the high/low pressure protections are triggered. Two variables are available for this purpose (user interface access):

• CfgEnvCtrl_BLDC.CstmEnv_DscgPMax custom envelope limit, maximum discharge pressure value (absolute kPa);

• CfgEnvCtrl_BLDC.CstmEnv_SucPMin custom envelope limit, minimum suction pressure value (absolute kPa).

The control system considers and uses envelope control:

• The minimum value between CfgEnvCtrl_BLDC.CstmEnv_DscgPMax and the default (or dynamic) value of the maximum discharge pressure limit (which depends on the selected compressor);

• The maximum value between CfgEnvCtrl_BLDC.CstmEnv_SucPMin and the default (or dynamic) value of the minimum suction pressure limit (which depends on the selected compressor);

 

Special functions: Siam compressor vapour injection

The vapour injection cycle using a liquid sub-cooler (economiser) is commonly used to improve performance in advanced low temperature applications. The introduction of a vapour injection scroll compressor exploits the cycle with economiser, using a single compressor where vapour is injected into the coil at intermediate pressure.

Input parameter VI_ModeOn is set to receive vapour injection activation status. When the function is on, the internal algorithm extends envelope limits and controls compressor rotation speed according to the manufacturer's specifications

Vapour injection valve monitoring is managed by the developer.

Principle

The vapour injection cycle with a scroll compressor is similar to a two-stage cycle with an intermediate cooling stage, but where the process uses a single compressor: part of the liquid coolant, on the high pressure side, passes through an expansion valve and then a plate heat exchanger that acts as a sub-cooler. Heated vapour is then injected at an intermediate pressure into the scroll compressor coil: additional subcooling increases the evaporator's cooling effect since enthalpy decreases at its introduction.

Additional subcooling increases evaporator enthalpy and thus, at the same capacity, a lower capacity compressor can be used.

This system can also be exploited to modulate compressor capacity if the cooling load is low: the vapour injection circuit can be bypassed using a solenoid valve, thus reducing cooling capacity.

COP gain

Cycle efficiency with a vapour injection scroll compressor is higher than that of a traditional single-stage compressor since the additional capacity due to subcooling is obtained with lower power consumption: vapour, produced in the subcooling process, is only compressed at intermediate pressure, which is higher than suction pressure.

 

Special Functions: circuit equalisation for compressor start

BLDC compressors can only be started when the pressure difference between suction and discharge is within the limit recommended by the manufacturer. When the compressor start is requested and the pressure difference is beyond the set threshold, pin CmdEqVlv is activated and compressor start is inhibited. This pin remains active until the pressure difference decreases under the threshold set for compressor start. If the driver EVD Start is enabled, the compressor starts only if the EVD driver is in standby mode EVDtoBLDC.RegStatus =3; ). Equalisation can also be performed with the electronic expansion valve. The module sends the cooling capacity percentage set CfgEnvCtrl_BLDC.StartUp_EVDEqualPreopen to pin BLDCtoEVD.CircCoolCap. If the pressure delta does not decrease within limit time CfgEnvCtrl_BLDC.StartUp_EVDEqualMaxT, the cooling capacity returns = 0 and compressor start is not enabled.

With electronic valve control, a cooling capacity not in line with valve opening is sent.

 

Special functions: test mode

A special function is included to debug the application on the simulator, accessible when the CAREL Power+ inverter is not found. "Test mode" is signalled when the "Incompatible inverter type" alarm is triggered.

Test mode may last up to 30 minutes. After this, the system is blocked and operations are only possible with the Power+ connected. To repeat tests, turn off system power before the 30 minutes have elapsed. Otherwise, the default settings must be reinstalled to repeat tests after the system was blocked (after the 30 minutes have elapsed).
