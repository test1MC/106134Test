General operations

"RotMng" works as manager of "MultiRot" circuits.

It can manage the rotation of up to 32 devices, divided in up to 32 circuits, following the FIFO logic. The block manages the circuits by the output array of structures "MngCircRot_RM" and receives feedback of max power notifications and requests status.

 

Request management

The activation of the devices is managed by the input variable "StepReq_RM" which takes the values from 0 to the number of total devices. When the request is managed as percentage, use the function block "PwrToStep" that calculates the number of devices to activate.

When the function block "PwrToStep" is used, connect to pin "N_Dev" the result of the multiplication of the number of circuit and the number of devices per circuit.

 

Number of circuits and devices

This block manages up to 32 circuits, the number of circuits "N_Circ_RM" and devices per circuit "N_DevCirc_RM" have to be set before the enabling of the block, or when the block is enable but the request is 0.

If the total number of devices exceeds 32, the block decrements the number of devices per circuit until the total number of devices fall in the maximum number of 32.

Equalized power of circuits

The input "EqualizedCircPwr_RM" allows the selection of two types of power distribution:

  • Equalized distribution
  • Grouped distribution

 

Equalized Distribution

The start and stop sequences are equally distributed between circuits.

Example

EqualizedCircPwr_RM = 0; N_Circ_RM = 2; N_DevCirc_RM = 3;

 

Grouped Distribution

The on and off sequences complete one circuit before continuing to the next.

Example

EqualizedCircPwr_RM = 1; N_Circ_RM = 2; N_DevCirc_RM = 3;

 

Timers

Devices timing are managed by "LdUpT_RM" and "LdDwnT_RM" input variables. For more information see "SingleRot" function block, paragraph "Timers".

 

Feedback from circuits

The block works with feedback of max power "MaxPwrCirc_RM" and request status "ReqCirc_RM" from the other circuits.

 

Management of circuits

"RotMng" controls the "MultiRot" with an array of structures "MngCircRot_RM"
Every structure contains fields that allow to manage a circuit: these fields are automatically parameterized by "RotMng" block according to its input pins (LoadUpT, LoadDownT, N_Dev, etc..).

 

Unavailable and Alarmed devices

Unavailable and alarmed devices have to be set in inputs of each "MultiRot" block: "RotMng" receives feedback from these blocks, interprets it and operates in the same way of the management of "SingleRot" function block's alarmed and unavailable devices.

 

How to implement more circuits
  1. Instantiate the required number of circuits (1 MultiRot block for 1 circuit)
  2. Connect in cascade the common input/output starting from the first to the last
  3. Set the same Enable block variable for all the blocks; set the correct IdxCirc for all the blocks (starting from 1)
  4. Instantiate the RotMng function block: connect to its input the 3 outputs of the last MultiRot block in cascade (ReqCirc_MR, MaxPwrCirc_MR and MinPwrCirc_MR)
  5. Connect the RotMng output (MngCircRot_MR) to the input of MultiRot blocks
  6. Set your own variables for all the other remaining parameters

For more information, see "Demo_EASY_ROT_LIB".
