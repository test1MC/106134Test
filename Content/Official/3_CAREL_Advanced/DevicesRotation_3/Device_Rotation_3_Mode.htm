Device_Rotation_3 - "Mode" variable

 

Last update 01/02/2016 | Native revision 09/02/2016

 

 

 

"Mode" parameter is inside the "CFG_DEVROT" structure of the "Conf" input pin.

Mode = 0

Normal rotation according to block configuration.

 

Mode = 1

Maintains always 2 devices ON per circuit, considering them as a single step.
This kind of control only works in equalised mode ("Conf.EqualizedCircPwr" = 1) with ON/OFF devices.

Mode = 2

Steps request mode: control the devices rotation request by setting the steps number in input "PwrReq" (E.g: 1 step = 0.1 request; 7 steps = 0.7 request)

 

Mode = 3

Maintains always 2 devices ON per circuit, but turns on the 3rd device in the same circuit.
