Pumps

Manages up to 2 pumps (ON/OFF or modulating) with thermal protection and flow alarm control, and integrated management of manual pump command

The Pumps function block permits the following functions:

  • With two pumps, pumps alternate manually or automatically to equally divide the work load and operating hours between each pump. Automatic rotation is generated:
    • At the end of a certain period of time
    • In the event of overloads (thermal overload triggered) or in lack of flow to one of the two pumps.
  • Pump overload management (thermal overload triggered). Fault signal and immediate pump stop. Alternation management with a second pump if present.
  • Flow switch management that controls the circulation of fluid inside the system (with Startup delays and different runs from each other). Controlled failure signalling (doesn't manage retries of no flow, the application needs to manage pump shutdown, so that the alarm reset management is made in c.strategy)
  • Modulating pump management (the FB input is a REAL 0-100%) in the output are present both BOOL (for the pump activation) and the request for the inverter.
General operations

Pumps is activated only if: En_PMP = TRUE

The output pins Pmp1_OnOff_PMP and Pmp2_OnOff_PMP take value TRUE to control pump activation.

The output pin PmpsActive_PMP is TRUE when at least 1 pump is active and water flow is present: this occurs with a delay from pump startup, defined by input values StartupDT_FlwChk_PMP and RunDT_FlwChk_PMP.
PmpsActive_PMP is necessary to enable operation of devices that require water flow, for example in Chiller applications to enable temperature control.

If there is no water flow (FlwSw_PMP = FALSE) during pump operation, after the RunDT_FlwChk_PMP delay, the output FlwSwAlrmPmp1/2_PMP is activated, depending on the active pump when the alarm is present, and PmpsActive_PMP is set to FALSE.

If pump changeover is defined by thermal overload, PmpsActive_PMP remains TRUE , allowing correct unit operation.

Pump alternation can be manual or automatic. Automatic alternation can be generated at expiry of the time or resulting from malfunctions.

The Pumps function block requires pump activation when PmpPwrReq_PMP >= 0.0 and En_PMP = TRUE, or when a manual pump activation request is present (Pmp1_Man_PMP <> 0 OR Pmp2_Man_PMP <> 0) and En_PMP = TRUE.

 

Pump rotation

With only one pump (PmpNo_PMP =1) the pump rotation function is not supported.

With two pumps (PmpNo_PMP =2) set RotT_PMP:

  • RotT_PMP > 0: automatic timed rotation (RotT_PMP represents the number of hours )
  • RotT_PMP = 0: rotation by manual switch (FrcPmpRot_PMP)

RotT_PMP affects the automatic rotation generated by malfunctions (thermal alarm) on a pump

 

Automatic timed rotation An operating time is set (hrs, pin RotT_PMP) after which rotation between pumps is forced.
Manual rotation

Pump rotation is activated by the rising edge of input pin FrcPmpRot_PMP.

the FB doesn't automatically set the variable connected to the pin to FALSE

Flow control

If flow control is enabled (En_FlwChk_PMP = TRUE) Pumps allows alarm management even in the event of no flow.

The alarm for no flow takes place with a delay respect to the starting of the pump defined by the input values StartupDT_FlwChk_PMP (at startup) and RunDT_FlwChk_PMP (at runtime). This is to ignore any changes in the water flow reported by pin FlwSw_PMP.

 

Pump overload

If an overload is found (Input pin OvldPmp1/2_PMP), the relevant pump is immediately stopped (PmpX_OnOff_PMP = FALSE).

If a second pump is present and it is not in alarm state, it is immediately activated keeping PmpActive_PMP = TRUE, so the unit can continue operating correctly; otherwise the output pin Al_PmpGroup_PMP is activated and PmpsActive_PMP = FALSE

 

Manual control

Manual management of the pump is activated when PmpX_Man_PMP > 0.

PmpX_Man_PMP = 0 è AUTOMATIC

PmpX_Man_PMP = 1 è Force OFF

PmpX_Man_PMP = 2 è Force ON (with request, in case of modulating pump, equal to PmpX_Man_PMP - 1)

Pumps behaviour in case of Force ON:

  1. If the pump was on it remains on, but possible rotation by time will be inhibited
  2. If the pump was off then it is activated and, at the same time, any other previously activated pump is turned off

Pumps behaviour in case of Force OFF:

  1. If the pump was on then it is turned off and, at the same time, the second pump is activated (if present and available)
  2. If the pump was off then the state doesn't change, but possible rotation by time will be inhibited .

If a pump is forced ON in manual mode, the absence of flow is not reported as an alarm, but only through PmpsActive_PMP = FALSE

 

Anti-block

This function is used to prevent physical or mechanical blockages caused by pump rust or encrustations due to prolonged pump inactivity.

Anti-block is enabled when En_Antiblock_PMP = TRUE.

If a pump is not started for more than seven days, Pumps automatically starts that pump for 30 seconds and then stops it. Anti-block has no effect on running pumps.
