General operations

PumpDown manages the pump down procedure in the circuit, using a liquid solenoid valve or by controlling the EEV valve.

 

The purpose of the pump down function is to reduce the quantity of refrigerant in the evaporator to limit the presence of liquid at the suction port during the compressor start-up phase.

 

It manages the pump down at Start and/or Stop, following these possible configurations:

  • NO pump-down (Typ_PMDW = 0);
  • pump down at Stop (Typ_PMDW = 1);
  • pump down at Start (Typ_PMDW = 2);
  • pump down at Start/Stop (Typ_PMDW = 3).

In case of pump down at Start,the only (PhRun_PMDW) phases executed will be 1, 2, 3 and then when the compressors goes off the phase rerturns to 1.

In case of pump down at Stop,the only (PhRun_PMDW) phases executed will be 1, 3, 4, 5, 6.

 

It using the EEV valve and/or the solenoid valve, the possible configurations are the following :

  • with EEVonly , with solenoid not installed (CfgVlv_PMDW = 2 and En_SolVlv_PMDW = 0);
  • with EEVonly , with solenoid valve installed (CfgVlv_PMDW = 2 and En_SolVlv_PMDW = 1);
  • with solenoid valve only (CfgVlv_PMDW = 1);
  • EEV and liquid solenoid valve (CfgVlv_PMDW = 3).

 

The pump down procedure ends when one of these conditions occurs:

  • Reaching the evaporating temperature (EvapTemp_PMDW < EvapTempThrsh_PMDW);
  • Overcoming the delta pressure, only for the pump down at start (DeltaP_PMDW >= MinDeltaP_PMDW);
  • Maximum duration time (MaxT_PMDW). In this case, (EndMaxT_PMDW) is signalled for one program cycle.

 

The FB uses two variables to manage compressor capacity. The 1st (GoToPmpDwnPwr_PMDW) is a command used, in the application, to set the compressor/s to the pump down capacity. The 2nd (PmpDwnPwrReach_PMDW) is a feedback, in input, to know when the compressor/s reach the pump down capacity, and thus can carry on with the procedure.

At the end of the pump down procedure (at Stop) a variable (ShOffRes_PMDW) is set to command deactivation of the compressor/s.

 

There is the possibility to add a delay (StartDT_PMDW) between phase 4 (decreasing compressor capacity ) and 5 (pump down at Stop). This is used to be sure that the compressor/s reach the pump down capacity before starting the actual pump down phase. It is useful when there is no accurate measurement of compressor capacity (i.e. stepless compressors).

 

It is possible to execute the pump down at Stop even in the case of non-serious alarms (LtAlrmAct_PMDW), if pump down at Stop is enabled. In this case the FB performs a fast pump down (from phase 3 goes to phase 5).

 

For correct behaviour, PwrReq_PMDW needs to be set based on running capacity . The value must be calculated based on the capacity of the active compressors (see the image in the examples).

The valve management in the pump down procedure at Start or Stop is different. The Start is manged by the solenoid valve. The Stop is managed by the EEV valve, because the gradual closure of the EEV prevents water pressure hammer problems, which may occur with immediate solenoid valve closure.

Examples
Example 1: circuit with 3 OnOff compressors

The FB receives some information in input on the capacity in the circuit and the probes:

  • PwrReq_PMDW calculated based on the capacity of the active compressors
  • CompsOn_PMDW from the status of the compressors
  • EvapTemp_PMDW from the probe reading
  • DeltaP_PMDW from the difference between discharge and suction pressure probe reading
  • MinDeltaP_PMDW from DeltaMinP of EnvPwr FB

And returns the command for the valve and the compressor/s:

  • Run_PMDW to Run_PMDW of DevAvbl F (in AND with the status of the compressor)
  • SolVlv_PMDW to command the digital output
  • EEV_FrcOff_PMDW to force the cooling capacity to zero
  • EEV_FrcStby_PMDW to DataIn_EVDEMB.EVD.Regulation.StandByEEV_OpEn.Valof EVD_Emb FB
  • EndMaxT_PMDW should be used to manage an alarm
