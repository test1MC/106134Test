General operations

The EbmpapstFan function block manages the ebmpapst® fan via Modbus protocol. It can manage:

  • Speed settings in rpm, 0-10V, 0-100%

  • Emergency function

  • Ramp-up/ramp-down times

  • Rotation direction

  • Voltage, current and power consumption reading

  • Work hours

  • Motor and alarm s tatus

  • Fan address change

  • Value read by sensors

for information on technical fan specifications, Modbus compatible models and all other details, visit ebmpapst website.

How to enable the emergency mode

In emergency mode, if the fan does not receive any Modbus command in a time span equal to EmergencyFunction_EBM.EmergLagT, it automatically changes the current speed to the speed indicated in variable EmergencyFunction_EBM.EmergSpeed.

To enable the emergency mode:

  1. Set the emergency speed value in EmergencyFunction_EBM.EmergSpeed

  2. Set the time after which the fan switches to the speed indicated by EmergencyFunction_EBM.EmergSpeed in EmergencyFunction_EBM.EmergLagT.

  3. Enable emergency mode using EmergencyFunction_EBM.En_Emerg_Funct=TRUE.

 

Changing the fan address using this function block

The fan address can be changed using an automatic procedure.

this re-routing only works if communication with the fan is already up.

Procedure steps are described below:

Automatic procedure:

  1. Change the value of Serial_InterfaceSettings_EBM.CfgProtocol.DevAddr and Serial_InterfaceSettings_EBM.FanSerInterSet.En_ChgAddr = TRUE. Next, the output parameters ChangeComParamStatus_EBM.StatusNewAddrProc takes value:

    • 1: Write new address
    • 2: Reboot fan firmware
    • 3: Wait offline
    • 4: Reset communication and wait online status;
    • 5:Reset procedure;

  2. When the value of ChangeComParamStatus_EBM.StatusNewAddrProc switches from 5 to 0, the fan is online and the address was correctly changed.

If communication is offline, changing Serial_InterfaceSettings_EBM.CfgProtocol.DevAddr variable change the device addres on the modbus protocol.
This puts communications online when a fan is replaced by another device with a different address (l.e.: fan replaced with a new one with address = 1).

Changing the fan address using this function block

The fan address can be changed via a Master device simulator for Modbus networks. In this case:

  • Set a new address in register 0xD100 (corresponds to 5350410, register name “Fan Address”)

  • Set register 0xD000 to 1 (corresponds to 5324810, register name “Reset”).
