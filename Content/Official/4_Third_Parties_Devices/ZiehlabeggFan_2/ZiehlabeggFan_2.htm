General operations

The ZiehlAbeggFan_2 function block controls a Ziehl-Abegg fan via Modbus protocol.

 

The following can be controlled with this function block:

  • Speed settings in “absolute speed” [rpm] or modulating interval “0 … 100.0%”.

  • Force shutdown via Modbus

  • Acceleration and deceleration time

  • Rotation direction

  • Power, voltage and current consumption reading

  • Motor and alarm status

  • Fan address change

  • Communication parameter change (parity, baudrate)

  • Emergency mode

The fan only starts rotating when the request exceeds the minimum activation threshold. The fan then starts to modulate according to the request.

If the request drops below the minimum speed, the fan continues to rotate at minimum speed and only stops when the request equals 0 (see Figure 1).

 

 

For further information on fan functions, available models and more, visit the ZIEHL - ABEGG website

Fan address change and communication parameters

The fan address, baudrate and parity settings can be changed using the ZiehlAbeggFan_2 function block.

This allows the fan communication parameters to be adpated to the connected network.

The change fan communication parameters procedure only starts if the device is online

 

Follow the steps below to change the address:

  • Set the SerialInterface_ZA.SerDevCfg.DevAddr variable to the new address
  • Set the SerialInterface_ZA.SerDevCfg.StartChgAddr variable to TRUE
  • Check the FanStatus_ZA.CommChgStatusvariable , it changes according to the progress of the procedure.

See the following diagram that describes the change address procedure:

Follow the steps below to change the baudrate and parity configuration:

  • Set the SerialInterface_ZA.FanSerialInterface.Baudrate and SerialInterface_ZA.FanSerialInterface.ParityConf variables to the new values
  • Set the SerialInterface_ZA.FanSerialInterface.StartCommChange variable to TRUE
  • Check the FanStatus_ZA.CommChgStatusvariable , it changes according to the progress of the procedure.

 

Emergency mode

With emergency mode you can specify how the fan will behave if the serial connection is not present

If not enabled, the fan will keep running at the same speed, otherwise after EmergLagT seconds it will go to the specified speed

a 0 second delay is not permitted, 0 rpm is permitted
