Project description
Features

This demo can be used to:

  1. Set the communication between a c.pCO and a KRIWAN gateway (both DP-Modbus Gateway and INT600 DM Gateway)
  2. See the information of INT69 device
  3. See the information of INT280 device
  4. See the information of INT600 device
  5. Test the communication between c.pCO and INT69, INT280 and INT600 devices
Proper operation

If the system is working in right way:

  1. No offline alarm are present
  2. With KRIWAN gateway disconnected the offline alarm happen for all the devices enabled
  3. No alarms from INT69, INT280 and INT600 devices
  4. If an alarm happens the memory error log is updated
  5. See the DP network status via INT600 masks
  6. See the compressor status via INT69 masks
  7. See the oil valve and oil status via INT280 masks
  8. See the simple or complete alarms table of INT69 or INT280 devices
    • Simple means that in the alarm mask it will be shown only the general error and then it is needed to go in the memory error alarm masks to see the last error;
    • Complete means that in the alarm mask it will be shown every kind of active alarm, then, for more information, it is possible to go in the memory error alarm masks to see the last error;
Main default settings

The following steps are necessary in order to use the demo:

  1. Set the communication frame configuration and address on the KRIWAN gateway, via dip switches
  2. Connect the c.pCO serial port to the KRIWAN gateway (Fieldbus2 for c.pCO / Fieldbus for c.pCO mini High End)
  3. Connect the KRIWAN gateway DP port to a KRIWAN INT69 or INT280 device (or both if you have the dedicated cable)
  4. Enable the available KRIWAN devices
Main settings
Set the same communication frame in c.pCO mask Baudrate, Stop bits and parity must be be set according to the KRIWAN gateway deepswtiches
Set the KRIWAN device address in c.pCO mask The value of the device address must be set according to the KRIWAN gateway dip switches
Enable a connected KRIWAN device Set to "Yes" the enable of an available KRIWAN device
Set to show simple or complete alarms table of INT69 or INT280 devices Set to "Simple" or "Complete" for the alarms table. In this way it possible to see or just the error status of INTxx device (Simple) or the alarms that has been triggered (Complete) (Default value = Simple)

 

Communication settings

Troubleshooting

Following are some tips for finding the possible causes of malfunction of this demo:

Problem Possible cause Suggested correction
Display is blank No power to controller Restore power supply
INT600 offline alarm Wrong wiring between c.pCO and INT600 device
 Cables must be checked
Wrong device address Set the right address according to the starting address set via dip switches (remember that the right address is starting address + 15)
DP network isn't shown correctly Try to reset the Configuration control dip switch to off and then to on, once the network will be shown as really connected
INT69 offline alarm Wrong wiring between INT600 and INT69 devices
 Cables must be checked
Wrong device address Set the right address according to the starting address set via dip switches (remember that the right address is starting address + its position in the network)
INT600 not ready to answer Modbus queries Wait for complete INT600 parametrization
INT280 offline alarm Wrong wiring between INT600 and INT69 devices
 Cables must be checked

INT600_LOG function block not enabled Check the variable for enabling the memory error
