General operations

The EbmpapstFan_2 function block manages the ebmpapst® fan via Modbus protocol. It can manage:

  • Speed settings in rpm, 0-10V, 0-100%

  • Emergency function

  • Ramp-up/ramp-down times

  • Rotation direction

  • Voltage, current and power consumption reading

  • Working hours

  • Motor and alarm status

  • Fan address change

  • Value read by sensors

for information on the fan technical specifications, Modbus compatible models and all other details, visit the ebmpapst website.

in Ebmpapst firmware < 5.00, Trasmission rate and parity configuration holding registers are not supported.

How to enable emergency mode

In emergency mode, if the fan does not receive any Modbus commands in a time span equal to FctEmerg_EBM.EmergLagT, it automatically changes the current speed to the speed indicated forvariable FctEmerg_EBM.EmergSpeed.

To enable emergency mode:

  1. Set the emergency speed value forFctEmerg_EBM.EmergSpeed.

  2. Set the time after which the fan switches to the speed indicated by FctEmerg_EBM.EmergSpeed in FctEmerg_EBM.EmergLagT.

  3. Enable emergency mode by settingFctEmerg_EBM.En_Emerg_Funct=TRUE.

Changing the fan parameters

When changing the value of some fan parameters, the VAR_OUTPUT WriteParamsReq_EBM takes the value TRUE to advise that device needs a load parameters command. In order to confirm and apply the changes, a positive edge is required at the LoadNewValues_EBM VAR_INPUT.

Changing the fan address using this function block

The fan address can be changed using an automatic procedure.

this re-routing only works if communication with the fan is already active.

The steps in the procedure are described below; automatic procedure:

  1. Change the value of SerInterfaceSet_EBM.CfgProtocol.DevAddr and SerInterfaceSet_EBM.FanSerInterfSet.En_ChgAddr = TRUE. Next, the output parameters ParamsChgComStatus_EBM.StatusNewAddrProc take the values:

    1. Write new address
    2. Reboot fan firmware
    3. Wait offline
    4. Reset communication and wait online status
    5. Reset procedure

  2. When the value of ParamsChgComStatus_EBM.StatusNewAddrProc switches from 5 to 0, the fan is online and the address has been correctly changed.

If communication is offline, changing the SerInterfaceSet_EBM.CfgProtocol.DevAddr variable changes the device address for the Modbus protocol.
This puts communication online when a fan is replaced by another device with a different address (l.e.: fan replaced with a new one with address = 1).

Changing the fan address using this function block

The fan address can be changed via a Master device simulator for Modbus networks. In this case:

  • Set a new address in register 0xD100 (corresponds to 5350410, register name “Fan Address”)

  • Set register 0xD000 to 1 (corresponds to 5324810, register name “Reset”).

Changing fan baudrate and parity bit using this function block

The fan baudrate and parity bit can be changed using a semi-automatic procedure.

this re-routing only works if communication with the fan is activeup.

The steps in the procedure are described below:

  1. Change the value of SerInterfaceSet_EBM.FanSerInterfSet.Baudrate and/or SerInterfaceSet_EBM.FanSerInterfSet.ParityConf;
    set     SerInterfaceSet_EBM.FanSerInterfSet.En_ChgCommInterfaceSet = TRUE;
    next, the output parameters ParamsChgComStatus_EBM.StatusNewAddrProc take the values:

    1. Write new address
    2. Load params required

  2. A write parameters command is now required (LoadNewValues_EBM)

  3. The device goes offline
  4. Manually change the serial line configuration and restart the application

changing the baudrate value also makes it necessary to align the command delay
