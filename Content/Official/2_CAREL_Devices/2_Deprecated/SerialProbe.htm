SerialProbe

Checks serial communications with the Carel probes via serial line

Block no longer in use and replaced by module SerialProble_2.

SerialProbe accesses all status variables and parameters in Serial Wall Probe devices via serial protocol. If enabled by the user, configuration parameters can be written from remote.

General operations

SerialProbe access all Serial Wall Probe device status variables and internal parameters via serial line through Carel Master or Modbus Master protocols. For this to happen it is necessary to set correctly the protocol editor in c.design tool.

The probe updating differential and the offset for the probes calibration can be manually modified by the user.

The user must wait at least 10 seconds after requesting changes to these parameters before turning off the system or the next overwrite is not guaranteed.

SerialProbe does not include unit of measure management which must be realized by developer outside the function block.

Enabling the Function Block

In order to correctly enable the function block, it's necessary:

  • instantiate a new SerialProbe function block
  • flag the entire function block in public variables selector as Protocol;
  • add the device into protocol line of protocol editor and share relative variables.

Serial probe calibration

To calibrate the external hardware connected to serial probe it's possible to set a temperature and humidity offset to add or subtract to the value read by probe.

The variables where to write the two offset are local variables of SerialProbe function block:

Variable Data types Description Default Limits
[SerialProbe].TempOffset INT Temperature offset in tenths of degree (°C) 0

-100..100
[SerialProbe].HumOffset INT Humidity offset in tenths of percentage (%) 0 -100..100
