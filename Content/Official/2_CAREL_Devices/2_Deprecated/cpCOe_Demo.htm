Project description

With this demo it is possible to:

  1. Manage a c.pCOe device
  2. Communicate with Modbus Master and Carel Master protocols
  3. Test all Inputs/Outputs of the expansion board
  4. Change address, parity and stop bits of c.pCOe using the wizard
Main default settings

The following steps are necessary in order to use the demo:

Main settings
Set 1 Connect a c.pCOe device to serial port. Set dip switches of c.pCOe
Set 2 When downloading the demo to a real device, select right configuration from c.design (c.pCO or c.pCOmini configuration)
Communication settings
Protocol Set Protocol variable (default 0=Modbus Master, 1=Carel Master)
Baudrate Set Baudrate_UI variable (0=9600, default 1=19200, 2=36400, 3=57600)
Stopbit Set StopBits variable (1 or default 2)
Parity Set Parity variable (default 0=None, 1=Odd, 2=Even)
Notes

In order to implement a full-functionality user interface, a support from strategy is required. Function block cpCOe_UI takes in charge to do some operations instead of masks, primarily for universal channels configuration. Is possible to build up a more simple user interface without using this block, for example if in your application fixed-direction channels are enough for your purposes.

In the demo is implemented a Wizard in order to help the procedure of changing address, parity and stop bits of the c.pCOe device (all these settings you cannot manage via dip switches).

There are two alarms available, beyond the retain memory alarm, are the offline alarm and the configuration error alarm.

The buzzer is set OFF by default in ST code of c.mask, but you can enable it if you need.
