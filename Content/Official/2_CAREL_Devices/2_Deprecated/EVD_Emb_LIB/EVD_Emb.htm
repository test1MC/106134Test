Structure of the application

The conversion function blocks are three: one for relative temperatures, one for absolute temperatures and one for pressures.

Conversion Function Blocks perform the conversion of unit of measure between variables used in user algorithm (collected in EVDEMB_DATA) and variables to display in the user interface. These are defined as local variable of Conversion Function Blocks.

The system configuration required:

  • two fixed parameters that define the unit of measure used in user algorithm; these are parameters because the logic is written for a specific unit of measure;
  • two variables with which select unit of measure used in user interface; these must be flagged on Public Variables Selector as DEV variables to be assigned by default; these are variable because you should change running time the unit of measure used;
  • if you use masks provided with Demo, you need a integer variable to manage change language.
EVD_Emb default value installation procedure from FB

With this function block is sufficient a boolean variable connected to InstDef pin to request default installation procedure. The procedure starts when EVD_Emb detects a positive edge at InstDef pin.

The procedure loads default values inserted by the user in DEV editor. Values not initialized in this way are automatically set to their factory default value (see paragraph Complete EVD_Emb parameter list).

This procedure can run if the module is enabled (En_EVD_Emb = TRUE).

The entire process takes only one cycle program.

To assign default value to a EVD_Emb parameter, it is necessary to:

  • Check the DEV variable attribute flag on Public Variables Selector (that allow to view variables in DEV editor)
  • open c.design > DEV editor
  • for each variable are visible two parameters: “.Dev” and “.EnDev”.
  • Insert the default value into “.Dev” and enable it putting TRUE in “.EnDev” in order to enable DEV value.

If you forget to assign TRUE to “.EnDev” the variable will set automatically to factory default value.

DEV editor must be filled before uploading the solution into device.

Filter Intensity

By c.design you can choose the filtering coefficient of analog input. Some test shown that for application with program cycle less than 1 second you should set Filter Intensity = 5; otherwise it’s recommended Filter Intensity = 6.

If you use Aux Regulation 5 (AC or chiller with BLD scroll compressor) you must set filter intensity to 6.

Data structure

All variables and parameters used by EVD_Emb are contained in a structure data called Data_EVD_Emb defined as global.

The data structure have four sublevels, everyone divided in other levels so variables are grouped by functionality.

For each variable are associated six parameters:

.Val

value of variable

.MskLimMax

maximum value that the variable can assume at user interface

.MskLimMin

minimum value that the variable can assume at user interface

.Err

is valorized by the variable integrity control.

0= no error; 1: out of maximum value; 2: out of minimum value

.Dev

contains the default value used in default value installation procedure

.EnDev

enables the use of the value contained in “.Dev” as default value

This parameters are grouped in a structure type. There are three type, one for each of three variable type used in algorithm:

  • REAL called EVDEMB_DATA_TYPEDESCR_REAL
  • INT called EVDEMB_DATA_TYPEDESCR_INT
  • BOOL called EVDEMB_DATA_TYPEDESCR_BOOL.

EVD_Emb synchronization

EVD_Emb parameters can only be set by algorithm using the variables provided by the function block, or by running the default installation procedure.

The output pin VlvReadyToGo = 1 signals the completed synchronization. Pin VlvReadyToGo is set when all the following conditions are met:

  1. after boot when valve is in Stand-by. This indicates that the driver forced closing and is ready for regulation; this time depends of the steps number of the valve (for Carel is approximately 10 sec. ).
  2. at the end of first program cycle after alignment between algorithm and EVD_Emb. This means that they are aligned;
  3. when there are no valve alarms or emergency closing is not enable;
  4. when the default installation procedure is not running.

Output pin VlvReadyToGo signals when Data_EVD_Emb.EVD.Variables.SH_UnitCoolCap.Val (> 0) can be sent to the driver with the meaning of the compressor operations can be start. In fact, the compressor should only start when the EVD_Emb is ready to control valve opening.

Managing the unit of measure

In EVD_Emb, pressure and temperature units of measure are completely unbind and independent so it’s possible to set °C/psig or °F/barg.

EVD_Emb works in °C/barg inside but it’s able to accept input variables in other unit of measure simply setting correctly BaseP and BaseTemp pin:

Pin Description Value
BaseP Pressure unit of measure used in algorithm(c.strategy application) FALSE:barg
TRUE:psig
BaseTemp Temperature unit of measure used in algorithm(c.strategy application) FALSE:°C
TRUE:°F

If you want to use different set to display variables in user interface and algorithm you need to use such conversion function block: EVD_EmbConverter_AbsTemp, EVD_EmbConverter_RelTemp, EVD_EmbConverter_P.

These converter function blocks manage:

  • the alignment between the same variable modified in user interface and algorithm;
  • the conversion based on the unit of measure chosen.

Each function block must be defined as global variable and its variable flags “User Interface” in “Public Variable Selector” table.

Modulating MOP Threshold

This function is typically used to ensure compressor operations with the allowed envelope zone.

To manage BLDC compressor operations is recommended to instantiate BLDC2EVD_Emb function block. The MOP threshold the driver uses for protection is communicated by input pin MOP_ThrshFastChg.

The function is on only if main regulation is enabled as control for BLDC compressors.

Valve regulation on superheating or discharge temperature

This function lets you monitor compressor discharge temperature.

To manage BLDC compressor operations is recommended to instantiate BLDC2EVD_Emb function block. The reference set point for these regulations is communicated via pins Dscg_SH_Set and DscgTempSet. Regulation is switched by input pin RegSubTyp.

The function is on only if main regulation is enabled as control for BLDC compressors.

EVD evolution working in DC direct current

To run EVD evolution in direct current (24VDC), set parameter Data_EVD_Emb.System.DC_SupplyEn. In this case, in the event of power outage, there is no forced emergency valve closing in EVD Evolution.

Service positioner

The pin ”En_ServicePosit” and ”ServicePosit_perc” allow the algorithm to change the valve position. It can be used for example to act on the valve in the event of active alarm.

The Manual Positioner available through mask is different from Service Positioner; they work at two different level: Manual Positioner has higher priority respect Service Positioner (if they are both enabled, the valve will be driven by Manual Positioner).

Mask

It's possible to recover the masks relative to EVD_Emb in Demo project (Demo_EVD_Emb).

The masks are collected in one loop and worksheet in order to make easy the import and integration in an existing project.

To know how import the packet, read "###_EVD_Emb_import_export_rules"

MaskEn groups boolean variables one for each regulation. When a regulation is enabled, the relative boolean is valorized TRUE so you can used it to enable masks, texts, special fields in c.mask.

Upload on target

Whenever you upload a new application with EVD_Emb inside on the devices or when you want to restore the unit configuration it’s strongly recommended to clean the permanent memory and cycle power the device.

In order to perform cleaning it's necessary open system mask by pressing “Alarm + Enter” for a few seconds; browse on Application > Wipe Retain and press Enter.

Use examples
Example 1: Probe backup utility and alarm management

The picture below shows en example of probe backup.

Temperature probe is normally equal to S2 value. The variable is equal to S2_Bck_Val when all the following conditions are met:

  • S2 Alarm is enabled;
  • the regulation is running running (this allows the regulation to start even if there's probes alarm due to equalization of pressure);
  • there’s an hardware error or S2 is out of bound.

S2 alarm is set when all the following conditions are met:

  • S2 Alarm is enabled;
  • there’s an hardware error or S2 is out of bound.

Example 2: Digital input management

It’s necessary to configure the IO Editor: associate a digital input to “DI1” boolean variable.

This digital input pilots the cooling capacity of EVD_Emb by Sel_INT function block switching between 0 to 100.

Example 3: Analog positioner

It’s necessary to associate a digital input to “DI1” boolean variable for the command enable of analog positioner.

An analog input is used to get the position in percentage. It can be set by choosing 0..10V type probe, putting the Min to 0.0 and Max to 100.0 so it can be connected directly to the input pin “ServicePosit_perc”of EVD_Emb.

Appendix 1: differences between Mod_EVDEVO and EVD_Emb

In this section we want to describe the most significant differences between Mod_EVDEVO and the new function block EVD_Emb.

Differences in functionality

EVD_Emb manages only one unipolar valve: references to the valve A or B does not exist anymore.

EVD_Emb contains the regulation logic whereas in Mod_EVDEVO the regulation is inside the firmware of driver valve.

Functionality present in Mod_EVDEVO but not implemented in EVD_Emb are listed below:

  • Digital Input management (it should be performed externally the function block);
  • Defrost delay management (it should be performed externally the function block since it depends on DI);
  • Analog positioner (it should be performed externally using the manual positioner);
  • Probe alarms management, such as backup probe and emergency valve position (it should be performed externally using the manual positioner).

This has been left to the developer that can implement it as best suit to the application.
