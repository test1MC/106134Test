General operations
Protocol configuration

cpCOe allows you to communicate with the expansion board through two protocols which can be selected via the input pin Protocol:

  • Modbus Master
  • Carel Master

The selection of the communication protocol of the expansion board is done via dip switches present on it (refer to the manual c.pCO). You can use them also to configure the electrical parameters (parity, and stop bits) or enter the set mode (refer to the section Set Mode).

The setting of the dip switches is interpreted only at the start of cpCOe board. This means that whenever the dip switches change position, the board must be power cycled for the new settings to take effect.

 

Address

The address of the expansion board is decided by the dip switch "Address" guaranteeing a range from 1 to 15: this range can be extended by the dip switch "Ext" (refer to the section Set Mode).

Mod_cpCOe calculates the address of the expansion card using the two pin DevAddr and AddrExt: the final address is given by the sum of the two.

Configuration

When a c.pCO is connected to a cpCOe, the FB sends the configuration of universal channels, digital outputs, slave timeout and safe mode delay: this configuration is saved in non-volatile memory of the expansion board. To properly configure the card, refer to section Configuration Error and Supervision Table.

Probes management

When a channel is set as active probe, the FB allows you to set the minimum and maximum limits for the linear conversion of the read value (calculation made on board of the cpCOe). For each probe type (active or passive) is possible to set an offset value for the correct calibration (calculation performed by FB). In addition, for each channel set as active probe, an alarm is triggered in case of probe breakage.

Filter intensity

When a universal channel is set as input, this parameter takes on different meanings according to the selected type:

TypeMeaningValues
40, 41Current in input0=50uA, 1=500uA, 2=TTL Levels
5Digital input type1=FAST, 2=NORMAL
All othersIntensity of the filter: the more the value is low (at the limit equal to 0), the more the output will change quickly in the event of rapid changes in input0..9

Update threshold

For each channel universal set as input, the transfer of data occurs when the value gained from the expansion card have a variation greater than or equal to the value defined by parameter UpdThrsh.

Digital outputs

The expansion card cpCOe has 6 digital outputs fully controllable via this FB. The state is decided by the input pin ValIn.DOuts, while with the following structure is possible to configure all parameters:

ParameterDescription
ChTypChannel type
TOutOnDelay in ms in which out is on (for types 2 and 3)
TOutOffDelay in ms in which out is off (for types 2 and 3)
EnSafeModeEnable safe mode
OfflinePatternPattern in case of offline

The state of the digital outputs can vary in 3 ways:

TypeBehaviour
0NORMALThe state is defined by the correspondent value of the array ValIn.DOuts
2CYCLICWhen the state defined by ValIn.DOuts is active, the relay remains open for a time TOutOn and remains closed for a time TOutOff cyclically
3DELAYEDWhen the state defined by ValIn.DOuts is active, the relay opens after TOutOn and turns off after TOutOff

When using DELAYED type, if TOutOff is lower than TOutOn, relay never closes.

Type 1 (DELAYED SYNC) is not available on cpCOe expansion board.

Safe mode

For the protection of loads driven by the cpCOe board , is necessary to establish the state of the outputs in the absence of communication with the c.pCO. This feature can be enabled via the configuration variable EnSafeMode.

When the cpCOe acknowledges to be offline from the master, outputs (both universal inputs set as output both digital outputs) are forced to the value determined by the variables OfflinePattern after a delay determined by SafeModeDT.

If the safe mode is not enabled, the value of output (whether universal or channel digital output) keeps the previous value when disconnecting.

Safe mode function won't be applied for universal channels set as inputs.

Configuration parameter SlvTimeout, which is the time the slave waits for a packet before to consider itself offline, should be greater than 0. In fact, 0 is a special value which means "never offline". If you set it to 0, cpCOe board will never enter the Safe Mode.

Set Mode

In order to change the address extension and the electrical parameters of the serial line (parity and stop bits) is necessary to bring the expansion board cpCOe in set mode by setting the dip switches "Address" and "Ext" to OFF. In this mode, the cpCOe communicates with fixed address 207, NONE as parity, and with 2 stop bits. Protocol and baudrate are instead kept consistent with the relative dip switches settings.

In this configuration, the input pin AddrExt and the configuration variable FrameCfg allow you to change the settings of the expansion card. Refer to the demo for an example of implementation in an application.

For more information, refer to the manual.

Address extension, parity and stop bits changed when cpCOe is not in set mode won't have any effect on it.

The maximum available extension for Modbus is 232, for Carel is 192.

Setting DevAddr 207 makes the FB ready to communicate to a cpCOe in set mode. In this way, you are free to change AddrExt without influence the serial line. Refer to Demo project to see how implement this procedure as automatic.

Configuration error

The expansion board cpCOe provides a configuration error when parameter is written outside its limits. If the error is different than 0, the FB report the problem to the user via the output pin CfgErr. Only the last error found will be shown: it will be care of the utilizer using the following code to identify from time to time the error and correct it until it becomes 0.

The output pin CfgErr returns the error code generated by the expansion board cpCOe. This code consists of 4 digits (XYYZ) whose meaning is given in the following table:

X (Category)YY (Identifier of the channel)Z (Parameter)
1Communication parameters01Wrong address extension (not in range 15-232)
2Wrong slave timeout (not in range 0-30000)
3Wrong safe mode delay (not in range 0-30000)
4Wrong frame configuration (not in range 0-5)
5Wrong master type (not 0 or 1)
2Universal channels1..101Wrong channel direction (not 0 or 1)
2Wrong number of samples for filtering (depends on channel type, see Filter intensity section)
3Wrong update threshold (not >0)
4Chip IO error (see Error variable related to the channel)
3Digital outputs1..61Wrong channel type (not 0, 2 or 3)
2Wrong TON (not in range 100-30000) only if DOut type is not standard (0)
3Wrong TOFF (not in range 100-30000) only if DOut type is not standard (0)
