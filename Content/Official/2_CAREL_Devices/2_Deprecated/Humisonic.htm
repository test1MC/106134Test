General operations

The Humisonic function block is used to manage the serial communication between the c.pCO and Humisonic humidifier. The function block manages communications via Modbus Master and Carel Master protocols.

Humisonic function block lets you set board variables linked to permanent settings or used by regulation.

 

The function block lets you manage the humidifier in two ways:

  • Proportional signal via serial: the device receives the percent production, calculated according to nominal production, using a value 0...1000, and works with a steam production proportionate to the sent value.
  • Humidity regulation: The function block sends the humidity set point to the device. The device changes production according to the set point and values read on the probes on the device.

Production is enabled by setting pin OnOffDev_HUMS to 1. If the pin is switched to 0, production stops, regardless of the request.

The function block also manages the device alarm functions, letting the user monitor any faults provided communications with the serial line are not down

 

Enabling the function block

To enable the function block set En_HUMS = 1.

 

Regulation with proportional value via serial

This method is enabled by setting pin DevSettings_HUMS.WorkingMode to 0 while production can be switched by setting the suitable value, in the 0...1000 interval, on pin SerialReq_HUMS.

Production can be disabled at any time by resetting OnOffDev_HUMS.

When the humidifier is controlled via serial, device probes are not used. Thus, no settings (from device or dedicated module mask) or regulation set point and hysteresis are required.

 

Humidity regulation

When this method is enabled, (DevSettings_HUMS.WorkingMode = 1) the humidifier runs proportional regulation using:

  • The set point in %rH sent by the c.pCO (pin DevSettings_HUMS.HumSetp)
  • The regulation hysteresis %rH sent by the c.pCO (pin DevSettings_HUMS.HumRegHysteresis)
  • The values read by the humidifier probes can be set using variables of pin DevSetting_HUMS.GenericPrb or the device display.

Please see the device manual for further information on the algorithm used for regulation.

When Humisonic humidifier is used in combination with the Humisonic function block, the type of regulation, the humidity set point and hysteresis, and the unit of measure can only be controlled from the c.pCO controller. If they are changed on the Humisonic display, they will be immediately overwritten by the values in the c.pCO. Other settings can be changed from both devices.

Generic Parameters Access

To permit full device control, the function block provides the chance to access parameters that are not directly managed in read and, when possible, in write. These parameters are not managed by default: the user must enable manually the communication by setting internal variable En_Generic_RW to 1. The function block will then automatically enable the related communication settings based on the protocol selected. Please see the manual for the complete table of parameters.

Communication settings

Modbus Master

For communications via Modbus Master protocol, download the default device templates from website KSA

Humisonic requires two device templates:

  1. Humisonic_Modbus_DefTable_1.0.0.0.txt - permits the communication with the parameters managed by default by the function block
  2. Humisonic_Modbus_GenericTable_1.0.0.0.txt - permits the communication with the parameters of Humisonic not handled by default by the function block

Carel Master

For communications via Carel Master protocol, download the default device template from website KSA

Humisonic requires one device template:

  1. Humisonic_Carel_ParamsTable_1.0.0.0.txt - permits the communication with all the parameters of Humisonic humifier
