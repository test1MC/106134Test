SerialProbe_2

Controls serial communications with Carel probes via serial line

SerialProbe accesses all status variables and parameters on Serial Wall Probe devices via serial protocol. If enabled by the user, the configuration parameters can be written from remote.

General operations

SerialProbe access all Serial Wall Probe device status variables and internal parameters via serial line using Carel Master or Modbus Master protocols. For this to happen, the protocol editor needs to be set correctly in c.design tool.

The probe updating differential and the offset for probe calibration can be manually modified by the user.

The user must wait at least 10 seconds after requesting changes to these parameters before turning off the system or the next overwrite will not be guaranteed.

Unit of measure

The Serial Probe as a device does not manage any conversion onboard, however but the function block allows the UoM to be managed via screen.
The input pin "UnitMeasure_msk_SP" has to be connected to the variable that manages the UoM for the entire application; the internal variable "UnitMeasure_msk" is used to manage the texts (°C, °F, etc..) for the unit of measure (see the image below).

"UnitMeasure_msk" must have the class "Temp_Msk_SP"

Enabling the Function Block

In order to correctly enable the function block, it's necessary to:

  • import the library with the program POU and alarms
  • import the Modbus Master template in c.design (import as template)
  • import the Worksheets in c.mask (import as template)

Serial probe calibration

To calibrate the external hardware connected to the serial probe, a temperature and humidity offset can be set and added to or subtracted from the value read by the probe.

The variables used to write the two offsets are input variables in the SerialProbe_2 function block:

Variable Data types Description Default Limits
TempOffset_SP INT Temperature offset in tenths of degrees (°C) 0

-100..100
HumOffset_SP INT Humidity offset in tenths of percent (%) 0 -100..100
