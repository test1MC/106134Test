General operations

The Humisonic_2 function block is used to manage serial communication between the c.pCO and Humisonic humidifier. The function block manages communications via Modbus Master and Carel Master protocols.

Humisonic_2 function block lets you set board variables linked to permanent settings or used by the control functions.

The function block can manage the humidifier in two ways:

  • Proportional signal via serial: the device receives the percentage of production, calculated according to nominal production, using a 0...100.0 value, and works with steam production proportionate to the value sent.
  • Humidity regulation: The function block sends the humidity set point to the device. The device changes production according to the set point and values read on the probes on the device.

Production is enabled by setting pin OnOffDev_HUMS to TRUE. If the pin is switched to 0, production stops, regardless of the request.

The function block also manages the device alarm functions, letting the user monitor any faults, provided communications with the serial line are not down.

 

Enabling the function block

To enable the function block set En_HUMS = TRUE.

 

Alarm management

Only one alarm at a time can be active: if more than one, the highest priority alarm code is indicated.

AlrmCode_HUMS Alarm type Priority
0 No alarms -
1 High humidity Low
2 Low humidity -
3 Tank maintenance required -
4 No production -
5 Faulty drain -
6 No water -
7 External signal not correctly connected -
8 Functional test not executed -
9 Autotest failure -
10 EEPROM High

 

Control with proportional value via serial

This method is enabled by setting pin DevSettings_HUMS.WorkingMode to 0, while production can be switched by setting the suitable value, in the range 0.0..100.0, on pin SerialReq_HUMS.

Production can be disabled at any time by resetting OnOffDev_HUMS.

When the humidifier is controlled via serial, device probes are not used. Thus, no settings (from device or dedicated module screen) or control set point and hysteresis are required.

 

Humidity control

When this method is enabled, (DevSettings_HUMS.WorkingMode = TRUE) the humidifier runs proportional control using:

  • The set point in %rH sent by the c.pCO (pin DevSettings_HUMS.HumSetp)
  • The control hysteresis %rH sent by the c.pCO (pin DevSettings_HUMS.HumRegHysteresis)
  • The values read by the humidifier probes can be set using variables on pin DevSetting_HUMS.GenericPrb or the device display.

Please see the device manual for further information on the control algorithm used.

When the Humisonic humidifier is used in combination with the Humisonic_2 function block, the type of control, the humidity set point and hysteresis, and the unit of measure can only be controlled from the c.pCO controller. If they are changed on the Humisonic display, they will be immediately overwritten by the values in the c.pCO. Other settings can be changed from both devices.

Generic Parameters Access

To permit full device control, the function block provides the possibility to access parameters that are not directly managed,both read and, where possible, write. These parameters are not managed by default: the user must manually enable communication by setting internal variable En_Generic_RW to TRUE. The function block will then automatically enable the related communication settings based on the protocol selected. Please see the device manual for the complete table of parameters.

 

Communication settings
Modbus Master

For Humisonic_2 communications via Modbus Master protocol, two different templates are available:

  1. DefTable permits communication with the parameters managed by default by the function block
  2. GenericTable permits communication with the Humisonic parameters not handled by default by the function block

 

Carel Master

For Humisonic_2 communications via Carel Master protocol, there is only one template:

  1. ParamsTable permits communication with all the Humisonic humifier parameters

Make sure in protocol editor that the Modbus line is entered first (ID = 0) and then the Carel line (ID = 1)
