How to implement a stable or beta library

 

Native revision 15/11/2016

INTRODUCTION

The BLDC_Comp_Mng and Inv_Power_Plus_3 modules have an independent versioning. They both are password protected.
The libraries are contained in the respective demos. The name of the demo is related to the module version.

 

BLDC LIBRARY

  1. Open the BLDC_Comp_Mng module. The password is required.

  2. Increase the version of the solution. If the version is a Beta, add the letter B after the version. Use the right mouse button on the solution to find the field to be modified.

  3. Increase the version of the project. If the version is a Beta, add the letter B after the version. Use the right mouse button on the solution to find the field to be modified.

  4. Increase the version of the module in Task_Main_Init.

  5. If entering a new compressor:

    • in the main_init task , Sel_Comp_Type module, create the variable with the new name of the compressor and itsallocation number.
    • in the main task, create a new folder with the new compressor parameters. Keep the sequence of the folder.
  6. Create a new project library with the name of the project and solution version.
  7. Save the BLDC_Comp_Mng module project in the new project library created. After first saving, repeat the process adding the password.
    This sequence is required to be sure it is password protected. Otherwise in the strategy the module may be shown with the lock but not be password protected.

Before publishing the library, make sure it is password protected.

 

Inv_Power_Plus_3 LIBRARY

  1. Open the Inv_Power_Plus_3 module. The password is required.

  2. Increase the version of the project. If the version is a Beta, add the letter B after the version. Use the right mouse button on the solution to find the field to be modified.

  3. If entering a new compressor:

    • in the ST subroutine, create a new folder with the new electrical parameters. Check the ST code. Keep the sequence of the folder.
    • in the main task, enter the st_call in a folder relating to the family of compressors. Increase the maximum number of compressors that can be selected in the Motor_Type variable.
      Keep the sequence of the folder.
    • in the mask editor, add the name of the new compressor.


  4. Create a new project library with the name of the project and solution version.
  5. Save the Inv_Power_Plus_3 module project in the new project library created. After first saving, repeat the process adding the password.
    This sequence is required to be sure it is password protected. Otherwise in the strategy the module may be shown with the lock but not be password protected.

Before publishing the library, make sure it is password protected.
