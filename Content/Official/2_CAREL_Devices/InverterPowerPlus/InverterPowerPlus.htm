General operations

InverterPowerPlus can only access Power+ devices via serial line with Modbus Master protocol Modbus Standard RTU (if all these conditions are true):

  1. Modbus Master protocol is correctly set via c.design

InverterPowerPlus acquires and displays Power+ device status variables and internal parameters. Access to main configuration parameters for editing is permitted from the User interface menus (see c.mask section).

Power+ parameters can be initialised to default parameters using the specific parameter. This operation is only possible if serial communication is active.

When communication is established, InverterPowerPlus acquires data that defines the Power+ connected: serial number, type of motor control, voltage and rated current. With the data acquired and the selected compressor , the module defines the optimal default parameters to configure the Power+. If the rated voltage of the Power+ is lower than that of the selected compressor, the module returns a message of electrical incompatibility, displayed on the screen used to load the default data and no operation is allowed.

InverterPowerPlus limits the maximum current from Power+ with auto setting of the nominal current value parameter to the maximum current required by the selected compressor. If the maximum current required by the compressor is higher than the nominal current of the Power+ connected, the "starting current" parameter is appropriately scaled in order to always guarantee the same Amperes during the compressor starting phase. The percentage of starting current is changed byInverterPowerPlus every time the "rated current" is changed.

When replacing the drive, InverterPowerPlus recognises the serial number of the new drive, verifies its configuration (voltage and current) and downloads the default parameters for the set compressor. This feature can be enabled by screen, and is disabled by default.

InverterPowerPlus manages and assigns different parameters for the same compressor depending on the connected drive. Check the "InverterPowerPlus compressor models supported" table for compatibility between compressors and drives. In the event of incompatibility, an error message is generated, displayed in the screen used to load the default data and the parameters cannot be downloaded. Compressor / drive compatibility can be forced manually from the special debug screen after entering the unlock code.

If compatibility is forced manually, CAREL declines all liability for anomalies or malfunctions of any kind.

InverterPowerPlus manages the motor safety control where the hardware will be designed. Communication with the motor safety control is via a serial line with Modbus protocol.

The Modbus address for communication with the motor safety control is automatically assigned by the module. The logical assignment is the next number to the address of the device.

InverterPowerPlus signals through pinout ReadyToGo_PWRP when the PSD is ready to receive the command. The pin is forced to 0:

  • when the PSD is offline
  • within 10 seconds of the first reading of parameters
  • downloading the default parameters

The run command is executed when the variable ReadyToGo_PWRP is true and there are no inverter alarms.

 

Crankcase heater management

InverterPowerPlus manages compressor crankcase heating. Heating is only possible with the compressor stopped and only if the PSD is not in alarm conditions.

The crankcase heater is managed by two variables:

  • ReqCrankCaseHeat_PWRP (pin input block): command that turns the heating cycle on and off.

  • CompCfgDefCstIn_PWRP.CrankCaseHeaterMode: Automatic, Manual ON and Manual OFF selection mode:

    • Automatic: Follows the ReqCrankCaseHeat_PWRP input pin request
    • Force ON: Always on with compressor stopped and PSD not in alarm conditions
    • Force Off: Always off.

The compressor start request takes priority over crankcase heating: with the heating cycle on and with compressor start request, crankcase heating is stopped. The run command is sent to the PSD 3 s after heating cycle stop.

Compressor stop with crankcase heating request on starts the crankcase heating cycle 3 s after compressor stop.

The value of the "HR65_CrankCaseHeaterCurrent" if less than 15% of the rated current of the compressor does not activate the crankcase heating process.

 

Alarm management

The InverterPowerPlus function block manages alarms acquired by Power+. Each single alarm reset can be set in two different ways: manual or semi-automatic. Semi-automatic reset attempts to automatically reset Power+ alarms, automatically sending the reset command. Manual reset is required when automatic attempts are timed out.

The default reset mode for all alarms is manual.

The "manual" or "semi-automatic" reset configuration is contained in the structure PWRP_AlarmMng_struct.

See also PWRP_AlarmMng_Struct

Each new alarm acquired by the InverterPowerPlus function block starts a time frame set in TimeThrs. Power+ alarms are automatically reset in this time frame. Reset attempts are incrementally counted on each new acquired alarm reset. The count is only decreased at the end of the time frame started by the alarm rising front. (See CONDITION 2)

The InverterPowerPlus function block signals Power+ alarm status and the automatic reset attempt in progress through variable Info_ResetRetryWaiting. Variable Info_ResetRetryExpired indicates the need to manually reset the alarm, this condition occurs when:

• the maximum number of attempts available in the TimeThrs time frame are timed out.

• the alarm is not automatically reset within the timeout period set by TimeThrs

• a single manual reset alarm is triggered

The manual reset command resets all automatic alarm reset attempt timers and counters.

The following conditions can occur:

• CONDITION 1: a single alarm event is triggered and the alarm is automatically reset within the control time.

• CONDITION 2: several alarm events are triggered and automatically reset within the control time.

• CONDITION 3: a single alarm event is triggered and, after a number of automatic reset attempts, cannot be reset within the timeout period since the alarm condition persists.

CONDITION 3 is an abnormal case and represents the attempt to automatically reset an alarm that does not reset. This could be due to an alarm that cannot be automatically reset.

The InverterPowerPlus function block attempts to reset the alarm event with cyclical resets. Cyclical resets are characterised by 10 s reset attempts and 30 s pauses.

After a Power+ offline condition and in alarm conditions, a single alarm reset impulse is automatically sent. This condition does not interact with the semi-automatic reset counters in any way.

Communication settings

Modbus Master protocols can be used for communications on the c.pCO – Power+ inverter with InverterPowerPlus.

The protocol type is unique for all application drivers. Set the driver network address with input pin Invadrr in InverterPowerPlus

Refer to the Power+ manual for models and relevant communication protocols.

For communications via serial protocol, download the device templates from the KSA website.

InverterPowerPlus requires the following templates:

 

Modbus Master
  1. Pwrp_Template_PSD : manages Power + control parameters.
  2. Pwrp_Template_Safe : manages Power + safe parameters. New PS2 hardware is needed.
