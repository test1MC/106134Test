General operations

The th-Tune function block manages the th-Tune room terminal via Modbus protocol: it can read/write terminal parameters for menu settings, humidity and temperature set point settings, time band settings and all those parameters typical of a room terminal.

The following are the main terminal features and possible application operations with the relevant paragraph on the pertinent topic:

Enabling the function block

To enable the function block, set En_THTN = TRUE.

The terminal must be turned on via pin WorkingModeIn_THTN = 1 or button .

Unit of Measure

The th-Tune terminal automatically converts the temperature to the required system and sends the value to the application. The th-Tune unit of measure system is set by UnitMeasure_THTN, while the unit of measure zone on the graphic interface is set by UnitMeasure_msk_THTN.

Temperature and Humidity
Th-Tune terminal probe values

Th-Tune terminal does not directly display temperature and humidity values (if present) from the probes installed on the terminal, but receives them both from the application. To display values received from the application in input, connect pins DisplayedDataSent_THTN.Temp and DisplayedDataSent_THTN.Hum to the required temperature and humidity values. In fact, the terminal displays the values via these two variables.

Temperature and Humidity setpoint

Variables DisplayedDataSent_THTN.TempSetP.Val and DisplayedDataSent_THTN.HumSetP.Val can be set both from the application and th-Tune terminal.

The last settings overwrite the previous ones.

To adjust set points from the terminal, set the maximum and minimum set point limits from the application (DisplayedDataSent_THTN.TempSetP.MinLim and DisplayedDataSent_THTN.TempSetP.MaxLim for temperature, DisplayedDataSent_THTN.HumSetP.MinLim and DisplayedDataSent_THTN.HumSetP.MaxLim for humidity).

Failure to set the minimum and maximum limits prevents the set point being set from the terminal.

Changing the set point from the terminal also affects the input pin variable.

Other temperature reading

An additional temperature reading can be displayed on the th-Tune terminal. Pin DisplayedDataSent_THTN.ExtPrb sends a second temperature reading from, for example, a serial probe.

All th-Tune function block pins that provide humidity and temperature values can be used to send any measurements to the terminal regardless of the type of probe that makes the reading.

Alarm conditions

The alarms associated with the th-Tune terminal are represented by the following variables:

Variable Description
Al_Offline_THTN If = TRUE means the peripheral, currently enabled, is disconnected from the network for some reason
Al_TempPrb_THTN If = TRUE signals an error on the internal th-Tune temperature probe
Al_HumPrb_THTN If = TRUE signals an error on the internal th-Tune humidity probe
Al_ClkBrd_THTN If = TRUE signals an error on the th-Tune clock board

Standard modes are always enabled by default and do not require settings. The application can:

  • select a mode (see paragraph Selecting an operating mode)
  • disable a mode if it won't be used using variables on input pin ModeVisualization_THTN

Icon pair modes are always disabled by default because they only used in special installations. The application must proceed as follows to work with combined modes:

  1. Define icon pair modes (See paragraph Defining an icon pair operating mode)

  2. Enable the icon pair mode using variables on input pin ModeVisualization_THTN

Next you can select an icon pair mode (see paragraph Selecting an operating mode).

Selecting an operating mode

The user needs to press the "mode" button on the th-Tune terminal or edit ModeStatusIn_THTN to select a standard or icon pair operating mode. The corresponding icon(s) is automatically displayed in the display area.

Icons only appear if the function block is enabled. Icon pair modes are always disabled by default. Therefore they must be first defined and enabled (see Defining an icon pair operating mode).

ModeStatusIn_THTN values that are not managed are ignored.

In order to align the operating mode with the application when it is changed on the th-Tune terminal, connect output pin WorkingMode_THTN to input pin WorkingModeIn_THTN. See the demo for implementation.

Clock
Display area Description

Current time/date appear in the dedicated display area.

Hold down the button to set the date/time.

The th-Tune terminal can use its internal clock board or the application clock. Variable TimeSet_THTN.ClkSrc sets which clock is used.

If the internal th-Tune clock is used, the hour, minutes and day of the week can be obtained from the terminal via variables on output pin DayTime_THTN.

Custom function button: configuration and enabling

A custom function can be linked to one of the four terminal buttons. One of the parameters returned by the function can be displayed in free parameter no.3 in the big area.

To set a custom function button, follow these steps:

Steps Description
1 Set free parameter 3: unit of measure, decimal point, acronym
2 Indicate the button that runs the function using the variables in variable KeysCfg_THTN.CstFctKeys
  Variable   Button
  KeysCfg_THTN.CstFctKeys Fan
  OnOff
  Clk
  Mode
3 Enable the function with KeysCfg_THTN.CstFctKeys.EnCstFct
4 Set KeysCfg_THTN.CstFctKeys.ShowVal to TRUE to display the value of free parameter 3 in the big area

The parameter returned by the function cannot be edited.
