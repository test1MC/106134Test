General operations

WirelessPrb manages the communication with a CAREL wireless probe.

The block needs as input the serial address of the probe and the configuration (see CFG_WPRB in the Structures List section for more information on the configuration).

The block automatically recognises the type of probe connected and enables the correct template for communication.

 

CAREL wireless probe types are:

Code Probe Type Probe description
62 EP SE Sensor for cabinets and cold rooms with 2 passive temperature probes and 2 DIn
63 BP SE Built-in sensor for cabinets with temperature probe
64 EP SE 2 Alternative code for EP SE sensor
65 SI Industrial sensor with temperature, humidity and light intensity probes
66 SA Ambient sensor with temperature and humidity probes
67 CI Sensor to be bound to energy meters with 2 counters, 2 passive temperature probes and 2 DIn
68 CI 2 Alternative code for CI sensor

Currently, only types SA (66) and SI (65) are supported. All other types will trigger an error (see Errors section).

Based on the type of probe connected, only a part of the input configuration is taken into consideration, and only some outputs have their values set.

For example, configuration parameters for the light sensor and the light value itself is ignored in case of SA probe, which doesn't have that sensor.

 

Errors

Err_WPRB explains the kind of the error which is preventing communication from working properly. It can take one of these values:

Error code Error description
0 No error
1 The type of the wireless probe connected is not supported by this Function Block
2 The device is online, but an error in the MainTable template is preventing the communication to work properly
3 The device is online, but an error in the CommonTable template is preventing communication from working properly
4 Address specified is not in range 16..126

This value doesn't come from the device, but from the function block.

 

Communication settings

For communications via serial protocol, download the device templates from website KSA.

WirelessPrb requires the following templates

Modbus Master

  1. WirelessPrb_CommonTable.txt - template for connecting and recognising the type of probe connected
  2. WirelessPrb_MainTableSA.txt - template for SA probe type
  3. WirelessPrb_MainTableSI.txt - template for SI probe type

Simply import the CommonTable and the MainTable templates related to the probe types in question.

If pCO connects to the probe successfully, the Online_WPRB output becomes TRUE and outputs are considered valid. If OfflineAlrm_WPRB is TRUE, values assumed by PrbVal_WPRB structure must be considered as invalid. Formore information about which type of error has occurred, check the value of Err_WPRB (see Errors section).
