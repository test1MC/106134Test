Project description

With this demo it is possible to:

  1. Manage a c.pCOe device
  2. Communicate with Modbus Master and Carel Master protocols
  3. Test all Inputs/Outputs on the expansion board
  4. Change address, parity and stop bits on the c.pCOe using the wizard
Main default settings

The following steps are necessary in order to use the demo:

Main settings
Set 1 Connect a c.pCOe device to the serial port. Set dipswitches on the c.pCOe
Set 2 When downloading the demo to a real device, select the correct configuration from c.design (c.pCO family controller configuration)
Communication settings
Protocol Set Protocol variable (default 0=Modbus Master, 1=Carel Master)
Baudrate Set Baudrate variable (9600, default 19200, 36400, 57600)
Stopbit Set StopBits variable (1 or default 2)
Parity Set Parity variable (default 0=None, 1=Odd, 2=Even)
Notes

In order to implement a full-function user interface, support from strategy is required. The cpCOe_2_UIMng function block handles management of certain operations in place of the screens, primarily for universal channel configuration. A simpler user interface can be built without using this block, for example if fixed-direction channels are enough for your purposes in your application.

 

A wizard implemented as a function block in the cpCOe_2 library in order to assist the procedure for changing address, parity and stop bits on the c.pCOe device (all of these settings cannot be managed via the dipswitches).

 

There are two alarms available: offline alarm and configuration error alarm.
