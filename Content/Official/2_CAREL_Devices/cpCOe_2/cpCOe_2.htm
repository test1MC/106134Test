General operations

cpCOe_2 manages communication between a c.pCO and a c.pCOe expansion card. In order to manage all universal channels and digital outputs arrays are used (length 10 for universal channels and 6 for digital outputs), so the number of pins is significantly reduced.

cpCOe_2_AddrChangeWizard and cpCOe_2_UIMng are two optional function blocks that help with the configuration of the c.pCOe.

cpCOe_2_AddrChangeWizard manages the wizard for changing the address extension of the c.pCOe (for the complete block documentation see "cpCOe_2_AddrChangeWizard").

cpCOe_2_UIMng manages the user interface for the I/O configuration of the c.pCOe (for the complete block documentation see "cpCOe_2_UIMng").

 
Protocol configuration

cpCOe_2 allows communication with the expansion card using two protocols, which can be selected via the Protocol input pin:

  • Modbus Master
  • Carel Master

The expansion card communication protocol is selected via the dipswitches provided (refer to the c.pCO sistema manual ). These can also be used to configure the electrical parameters (parity and stop bits, or enter Set Mode ).

The setting of the dipswitches is interpreted only when the c.pCOe card starts. This means that whenever the dipswitches change position, the card must be power cycled for the new settings to take effect.

 

Address

The address of the expansion card is defined by the "Address" dipswitch, guaranteeing a range from 1 to 15: this range can be extended by the "Ext" dipswitch (refer to the Set Mode section ).

cpCOe_2 calculates the address of the expansion card using the two pins DevAddr_CPCOE and AddrExt_CPCOE: the final address is given by the sum of the two, which can be read from the output pin DevAddrExt_CPCOE.

 

Configuration

When a c.pCO controller is connected to a c.pCOe, the FB sends the configuration of the universal channels, digital outputs, slave timeout and safe mode delay: this configuration is saved in non-volatile memory on the expansion card. To properly configure the card, see "Configuration error" and "cpCOe Supervisor" .

Probe management

When a channel is set as an active probe, the FB allows the minimum and maximum limits to be set for the linear conversion of the value read (calculation made on-board the c.pCOe). For each probe type (active or passive), an offset value can be set for correct calibration (calculation performed by the FB). In addition, for each channel set as an active probe, an alarm is triggered if the probe is broken.

Update threshold

For each universal channel set as an input (ChDir=0), data is transferred when the value acquired from the expansion card varies by greater than or equal to the value defined by parameter UpdThrsh (this is true only for Carel Master protocol because c.pCOe sends a new value only when the value changes, with Modbus protocol c.pCOe sends a new value even if the value is not changed). This can be useful if there are fluctuations in the value read by the probe.

Digital outputs

The c.pCOe expansion card has 6 digital outputs, fully controllable via this FB. The state is decided by the input pin ValIn_CPCOE.DOuts, while the following structure (CFGDOUT_CPCOE) can be used to configure all the parameters:

ParameterDescription
ChTypChannel type (0:standard, 2:cyclic, 3:delayed)
TOutOnDelay in ms in which out is on (for types 2 and 3)
TOutOffDelay in ms in which out is off (for types 2 and 3)
EnSafeModeEnable safe mode
OfflinePatternPattern if offline

 

The state of the digital outputs can vary in 3 ways (more details can be found in c.design help, topic "I/O Editor" chapter "Configure digital outputs" Table 2):

TypeBehaviour
0STANDARDThe state is defined by the corresponding value of the array ValIn.DOuts
2CYCLICRelay closing and opening cycle. For TOutOn milliseconds the relay is closed and for TOutOff milliseconds the relay is open cyclically
3DELAYEDWhen ValIn.DOuts=1 the relay is closed after TOutOn milliseconds and open after TOutOff milliseconds

 

When using DELAYED type, if TOutOff is lower than TOutOn, the relay never turns on.

Type 1 (DELAYED SYNC) is not available on the c.pCOe expansion card.

 

Safe mode

For protection of the loads managed by the c.pCOe card, the state of the outputs needs to be established in the absence of communication with the c.pCO. This feature can be enabled via the configuration variable EnSafeMode.

When the c.pCOe detects that it is offline from the master, the outputs (both universal inputs set as outputs and digital outputs) are forced to the value determined by the variables OfflinePattern after a delay determined by SafeModeDT.

If safe mode is not enabled, the output value (whether universal channel or digital output) keeps the previous value when disconnecting.

 

 

Safe mode function is not applied for universal channels set as inputs (ChDir= 0).

Configuration parameter SlvTimeout, which is the time the slave waits for a packet before considering itself offline, should be greater than 0. In fact, 0 is a special value which means "never offline". If this is set it to 0, the c.pCOe card will never enter Safe Mode.

 

Set Mode

The 'Set mode' allows the 'address extension' and the communication parameters of the serial line (parity and stop bits) to be changed. To set the c.pCOe expansion card in 'Set mode', set the dip switches "Address" and "Ext" to OFF. In this mode, the c.pCOe communicates with fixed address 207, NONE as parity, and with 2 stop bits. Protocol and baudrate are on the other hand kept consistent with the corresponding dipswitch settings.

In this configuration, the input pin AddrExt_CPCOE and the configuration variable FrameCfg allow the settings of the expansion card to be modified. Refer to the demo for an example of implementation in an application. For this purpose, the cpCOe_2_AddrChangeWizard block is used (for the complete block documentation see "cpCOe_2_AddrChangeWizard").

For more information, see the c.pCO sistema manual.

 

Address extension, parity and stop bits changed when c.pCOe is not in set mode won't have any effect.

The maximum available extension for Modbus is 232, while for Carel it is 192.

Setting DevAddr_CPCOE 207 makes the FB ready to communicate with a c.pCOe in set mode. In this way, AddrExt_CPCOE can be changed freely, without affecting the serial line. Refer to the Demo project to see how to implement this procedure automatically.

 

Configuration error

The c.pCOe expansion card provides a configuration error when a parameter is written outside its limits. If the error is other than 0, the FB reports the problem to the user via the output pin CfgErr_CPCOE. Only the most recent error found will be shown: the user will be responsible, using the following code, for identifying and correcting the errors so that this returns to 0.

Communication settings

For communications via serial protocol, the device templates are inside the library, simply check that the template folder for a project that uses the cpCOe_2 library has the following templates.

Modbus Master

There are three templates, plus one only for enhanced model, that include variables with different addresses. Therefore, it is mandatory to use them all. The system is made to optimise the polling time when the device is in the normal operating phase; in this phase any unnecessary templates are disabled.

  1. cpCOe_2_MBM_StartUpTable.txt - startUp table, used when the c.pCOe is online

  2. cpCOe_2_MBM_CfgTable.txt - IO config table, used when the c.pCOe is online after the StartUp table

  3. cpCOe_2_MBM_IOTable.txt - working table

  4. cpCOe_2_MBM_ExtPositTable.txt - valve management table (enhanced model only)

Carel Master

  1. cpCOe_2_CMST.txt

 

Devices works in Modbus slave protocol, not in Carel Slave protocol (enhanced model only)

 

Polling Time Threshold

(enhanced model only)

When the polling time of the Modbus network exceeds 10 seconds the output ThrshWarnPollTime_CPCOE switches to TRUE.

This control is performed only with c.pCOe Enhanced model, with Basic model this variable is disabled and stays FALSE.
