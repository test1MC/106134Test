Project description

With this demo it is possible to:

  1. Manage a c.pCOe device
  2. Communicate with Modbus Master protocol
  3. Test all Inputs/Outputs on the expansion board
  4. Test EEV valve management on the expansion board
  5. Management of probe readings
  6. Change address, parity and stop bits on the c.pCOe using the wizard
Main default settings

The following steps are necessary in order to use the demo:

Main settings
Set 1 Connect a c.pCOe device to the serial port. Set dipswitches on the c.pCOe
Set 2 When downloading the demo to a real device, select the correct configuration from c.design (c.pCO family controller configuration)
Communication settings
Protocol Only Modbus Master available
Baudrate Set Baudrate variable (default 19200, 36400, 57600)
Stopbit Set StopBits variable (1 or default 2)
Parity Set Parity variable (default 0=None, 1=Odd, 2=Even)

 

Input Probes

In main mask there is a variable to select whether the probes are connected to c.pCOe or c.pCO mini, default connect probes at c.pCOe UnivCh.

The probes default settings are explain in the table below.

 

Probe EVD_Emb UnivCh c.pCOe Enh UnivCh c.pCO mini Type
S1 UnivCh1 UnivCh1 43 - 0,5..4,5 V [min:-1.0, max:9.5]
S2 UnivCh2 UnivCh2 0 - Carel NTC
S3 UnivCh3 UnivCh3 43 - 0,5..4,5 V [min:-1.0, max:9.5]
S4 UnivCh4 UnivCh4 0 - Carel NTC

 

EVD_Emb configuration
Control Centralized Cabinet or Cold Room
Valve Carel E2Vu
SH Set 11.0

 

Valve

An unipolar EEV valve must be connect to c.pCOe Enhanced connector. It need not necessarily be CAREL EV2u type, it has set in "EVD_Emb_2" Function Block.

Notes

In order to implement a full-function user interface, support from strategy is required. The cpCOe_2_UIMng function block takes on the management of certain operations in place of the screens, primarily for universal channel configuration. A simpler user interface can be built without using this block, for example if fixed-direction channels are enough for your purposes in the your application.

 

A wizard implemented as a function block in the cpCOe_2 library in order to assist the procedure for changing address, parity and stop bits onthe c.pCOe device (all of these settings cannot be managed via the dipswitches).

 

The probes can be connected at c.pCOe or c.pCO mini Universal channels, see 'Input Probes' paragraph.

 

The EVD_Emb_2 Function Block is used in valve management, the variable ExtPosit_1 (structure TEEV), which contains the valve management parameter, is passed to cpCOe_2 Function Block.

EVD_Emb_2 Function Block can be placed in other POUs in application.

 

Example of Use
Manual positioning

Manual control of the valve position.

Steps:

  1. Connect the system;
  2. When the c.pCOe is online, press Prg;
  3. Select "2. EVD_Emb";
  4. Search "Manual positioning" mask;
  5. Enable and set the position of valve, the valve follows the request.

 

Regulation

Simulation of probes value and valve management operate to reach SuperHeat setpoint.

Steps:

  1. Connect the system;
  2. Connect to c.pCOe Enhanced Universal Channel a simulation of probes' value.
  3. Set real values (ex. S1: 6.0 bar, S2: 25.0 °C, S3: 6.0 bar, S4: 25.0 °C)
  4. In main loop set Cooling Capacity= 100%
  5. Press Down and in mask see the SH and valve position variation when the probes' value change.
    When the SH is less than setpoint (SH Set= 11.0) the valve close, when it is greater than setpoint the valve open.

 

Troubleshooting
  • Probe values is not correct: check the type of channel and limits.
  • The valve does not respond: check the valve type in menu "2. EVD_Emb".
  • c.pCOe isn't online: check address, protocol (Enhanced model work only in ModBus network), baudrate (19.2k, 38.4k, 57.6k and NOT 9.6k, parity and stop bits.
