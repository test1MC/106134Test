Project description

Project aims are to:

1. Show how EVD_EVO can be managed inside a c.suite project.

2. Test the main device functions

With this demo, it is possible to:

  1. Read probe values and see the circuit overview

  2. Allow the driver to start control, by setting a Cooling Capacity in the main loop

  3. Understand how to use alarms

  4. Understand the default installation logic both by DEV editor and c.strategy (by input pin "InstDefIn_EVD")

  5. Import/Export parameters set as Retain with a DEV value

Getting started

In order to successfully communicate with the EVD driver and allow the valve to control, verify that:

  • "DrvReadyToGo_EVD" goes to TRUE (after more or less 1 minute)
  • There are no communication alarms
  • Probe values are correct
  • The overview screens show correct values
  • Cooling Capacity in the main loop is > 0
Default configuration

"Demo_EVD_EVO_2" provides templates for the default configuration both from DEV and c.strategy.

Also Initial values are present in the Demo POU, to ensure that the EVD_EVO_2 starts from a clean situation.

Manual positioning

Manual control of the valve position.

Steps:

  • Connect the system
  • When the driver is online, press Prg
  • Select "Manual mng."
  • Enable and set the valve position: the valve follows the request
Unit of Measure management

As shown in the image , EVD and c.pCO always communicate using International System units; the final user can set their own unit of measure for display only.

Troubleshooting
  • Configuration error: see "How to debug a Configuration error" in EVD_EVO_2 documentation. This error can be caused by a wrong Super parameter configuration or by a parameter out of range. Also check the default assignment.
  • Valve is not controlling: check Cooling Capacity value in the main loop and if "DrvReadyToGo_EVD" = TRUE
  • High polling time: probably there's an exception in the line (from the EVD driver) caused by a parameter out of range. The exception usually appears with a Configuration Error, check "How to debug a Configuration error" in EVD_EVO_2 documentation
  • EVD driver offline: check address (both on c.pCO and EVD driver), baudrate, parity and stop bits. These variables must be retain.
  • Some parameters have the value #####.#: probably the UoM has been set to Imperial System (USA). #####.# means that the value is -777-7 (parameter disabled)
  • Not able to enter the "Configuration" menu: to enter "Configuration",the Function Block needs to be enabled in the main loop
