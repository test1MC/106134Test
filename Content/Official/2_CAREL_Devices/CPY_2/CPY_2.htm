CPY_2

The CPY_2 function block is used to manage serial communication drivers between the Cc.pCO and the CPY board for the KUE humidifier kit.

General operations

CPY_2 manages communication between a c.pCO and a CPY board for the KUE humidifier kit.

It manages:

  • Steam production request
  • Optional function configuration
  • CPY configuration
  • Alarm reset
  • Cylinder life reset
  • Manual operations
  • Change of humidifier model

 

For a full list of board variables available for the network, see chapter Board control via network, in the board user manual.

For easier board and function block variable identification, the prefix “Ax_By” has been added to each identifier, where:

  • “Axx”: indicates the type of data. (A = analogue, I = integer, D = digital) and the index (xx) of the supervised variable (from the CPY Modbus table)
  • “By”: indicates the corresponding parameter number in CPY

Example: I05_B1_OptionalFunctions is the integer variable with address 05 that corresponds to CPY parameter B1.

 

When this Function Block is used the model specified to the block is forced to CPY.

 

Communication settings

Following are the standard CPY configuration settings for serial communications with C.pCO

  • C3 – serial address (depends on the code, usually 1 or 2)
  • C4 – baud rate (0 = 9600; 1 = 19200)
  • C5 – supervisor: frame (bits per character, parity stop bits; according to programming)

To open these settings from the CPY terminal, press and hold down “ENTER” in the main screen for two seconds, enter “77” and scroll the screens to view the required settings. Reboot the CPY controller to save the settings.

Changes to serial connection settings only apply after the CPY is rebooted.
