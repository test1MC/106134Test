Project description

Project aims are:

  1. Show how EVD_Emb_2 can be managed inside a c.suite project.
  2. Test all the device’s functions

Demo/Test application for EVD_Emb_2:

  • Management of measurement system conversions
  • Management of probe readings
  • Example of use of alarms
  • Default installation logic by DEV editor
  • Common functions are grouped into the same strategy page in order to allow a better focus and prepare that page for exporting.

The following steps are necessary in order to use the demo:

S1_EVDEMB 43 - 0,5..4,5 V [min:-1.0, max:9.5]
S2_EVDEMB 0 - Carel NTC
S3_EVDEMB 43 - 0,5..4,5 V [min:-1.0, max:9.5]
S4_EVDEMB 0 - Carel NTC

Circuit number 1

Mode Ac or Chiller
Common parameters No

Valve Carel E2Vu
Getting started

In order to successfully communicate with the EVD embedded driver and allow the valve to regulate, verify that:

  • "VlvReadyToGo_EVDEMB_1" goes to TRUE (after more or less 1 minute)
  • There are no alarms
  • Probe values are correct
  • The overview screens show correct values
  • Cooling Capacity in the main loop is > 0
Default configuration

"Demo_EVD_Emb_2" provides templates for the default configuration both from DEV and c.strategy.

Also Initial values are present in the Demo POU, to ensure that the EVD_Emb_2 starts from a clean situation.

Manual positioning

Manual control of the valve position.

Steps:

  • Enable the function block in "2. EVD_Emb" menu
  • When the driver is ready (Setup done is set to YES) go in the "Manual positioning" mask
  • Enable the manual positioning
  • Set the position of valve: the valve follows the request
Unit of Measure management

EVD_Emb_2 only manages the International System set of units of measure (°C/barg).

Notes

The demo is designed for operation with one valve. The external positioner is ignored, for more infomation about this functionality check the Demo_EVD_Emb_cpCO_Enh application.
