General operations

EVD_Emb_2 manages the EEV service on the c.pCOmini board, according to EVD Evolution algorithm. The block lets the OS run all process control.

The block is designed for operation with two valves, even if the hardware manages just one valve. The second valve cannot use it

in some applications, superheat set point parameters (SH) and PID, LOP, MOP and LowSH settings may frequently vary. To correctly use these parameters, see paragraph Frequently changing parameters.

for more details on control and use of the probes, see the EVD evolution device manual +0300005EN.

Upload on target: whenever uploading a new application with EVD_Emb_2 inside to the devices or when wanting to restore the unit configuration, it’s strongly recommended to clear the permanent memory and power cycle the device.
To clear the memory, open the system screen by pressing “Alarm + Enter” for a few seconds; browse to Application > Wipe Retain and press Enter.

EVD_Emb_2 default value installation procedure from FB

With this function block, it is sufficient to have a Boolean variable connected to InstDef_EVDEMB pin to request the default installation procedure. The procedure starts when EVD_Emb_2 detects a positive edge at InstDef_EVDEMB pin.

The procedure loads the default values entered by the user in DEV editor. Values not initialised in this way are automatically set to their factory default value (see "Appendix 1: complete EVD_Emb_2 parameter list and Mod_EVDEVO crossreference").

This procedure can run if the module is enabled (En_EVDEMB = TRUE).

The entire process takes only one program cycle.

To assign the default value to a EVD_Emb_2 parameter, proceed as follows:

  • Check the DEV variable attribute flag in the Public Variables Selector (this allows variables to be viewed in DEV editor)
  • open c.design > DEV editor
  • for each variable, two parameters are visible: “.Dev” and “.EnDev”.
  • Enter the default value into “.Dev” and enable it by setting “.EnDev” to TRUE in order to enable the DEV value.

Example


If you forget to assign TRUE to “.EnDev” the variable will be automatically set to the factory default value.

DEV editor must be completed before uploading the solution into device.

Filter Intensity

In c.design you can choose the filtering coefficient for the analogue input. Tests show that for applications with a program cycle less than 1 second you should set Filter Intensity = 5; otherwise it is recommended to set Filter Intensity = 6.

If you use Aux Regulation 5 (AC or chiller with BLDC scroll compressor) you must set filter intensity to 6.

Data structure

All variables and parameters used by EVD_Emb_2 are contained in a data structure called Params_EVDEMB defined as global.

The data structure has four sublevels, each divided into other levels so that variables are grouped by function.

Each variable is associated with six parameters:

.Val value of the variable
.MskLimMax maximum value that the variable can take on the user interface
.MskLimMin minimum value that the variable can take on the user interface
.Err the value is set by the integrity control variable .
0= no error; 1: out of maximum value; 2: out of minimum value
.Dev contains the default value used in the default value installation procedure
.EnDev enables the use of the value contained in “.Dev” as default value

These parameters are grouped into a structure type. There are three types, one for each of the three variable types used in the algorithm:

  • REAL called DATA_TYPEDESCR_REAL_EVDEMB
  • INT called DATA_TYPEDESCR_INT_EVDEMB
  • BOOL called DATA_TYPEDESCR_BOOL_EVDEMB

 

EVD_Emb_2 synchronisation

EVD_Emb_2 parameters can only be set by algorithm using the variables provided by the function block, or by running the default installation procedure.

The output pin VlvReadyToGo_EVDEMB = 1 signals completed synchronisation. Pin VlvReadyToGo_EVDEMB is set when all the following conditions are met:

  1. after boot when valve is in Standby. This indicates that the driver has completed forced closing and is ready for control; this time depends on the number of valve steps (for Carel this is approximately 10 sec. ).
  2. at the end of first program cycle, after alignment between algorithm and EVD_Emb_2. This means that they are aligned;
  3. when there are no valve alarms or emergency closing is not enabled;
  4. when the default installation procedure is not running.

Output pin VlvReadyToGo_EVDEMB signals when Params_EVDEMB.EVD.Variables.SH_UnitCoolCap.Val (> 0) can be sent to the driver with the meaning that compressor operation can start. In fact, the compressor should only start when EVD_Emb_2 is ready to control valve opening.

 

Managing the unit of measure

EVD_Emb_2 only manages the International System set of units of measure (°C/barg).

Modulating MOP Threshold

This function is typically used to ensure compressor operation within the allowed envelope zone.

To manage BLDC compressor operation, it is recommended to instantiate the BLDC2EVD_Emb_2 function block. The MOP threshold that the driver uses for protection is communicated by input pin MOP_ThrshFastChg.

The function is on only if the main control is enabled as control for BLDC compressors.

 

Valve control based on on superheat or discharge temperature

This function lets you monitor the compressor discharge temperature.

To manage BLDC compressor operation, it is recommended to instantiate the BLDC2EVD_Emb_2 function block. The reference set point for these types of control is communicated via pins Dscg_SH_Set and DscgTempSetRegulation and is switched by input pin RegSubTyp.

The function is on only if the main control is enabled as control for BLDC compressors.

EVD evolution working in DC direct current

To run EVD evolution in direct current (24VDC), set parameter Params_EVDEMB.System.DC_SupplyEn. In this case, in the event of a power outage, the force emergency valve closing on EVD is only performed on hardware that supports this.

Service positioner

The pins ”En_ServicePosit_EVDEMB” and ”ServicePosit_perc_EVDEMB” allow the algorithm to change the valve position. This can be used for example to act on the valve in the event of an active alarm.

The Manual Positioner available from the screen is different from the Service Positioner; they work at two different levels: Manual Positioner has higher priority than Service Positioner (if they are both enabled, the valve will be driven by Manual Positioner):

  • each positioner mode freeze the PID control
  • during Manual or Service positioner the machine control is freezed
  • if the LOP protection was active, then it stays in LOP protection until the positioner will be disabled
  • if the positioner is enabled without any protection, then no protection start until the protection will be disabled

The Manual and Service positioner are very similar, but with these differences:

  • Manual has high priority than Service
  • Manual is in steps, Service is in % of opening
  • Manual is usable via dedicated mask, so it is possible to pilot the valve by PGD also if the machine is in stop
  • Service is usable via c.strategy algorithm, for example, to pilot the valve in case of a probe alarm

Examples
Example 1: Probe backup utility and alarm management

The picture below shows en example of probe backup.

Temperature probe is normally equal to the value of S2. The variable is equal to S2_Bck_Val when all the following conditions are met:

  • S2 Alarm is enabled;
  • control is running (this allows control to start even if there is a probe alarm due to equalisation of pressure);
  • there is a hardware error or S2 is out of bounds.

S2 alarm is set when all the following conditions are met:

  • S2 Alarm is enabled;
  • there is a hardware error or S2 is out of bounds.

Example 2: Digital input management

The IO Editor needs to be configured: associate a digital input to “DI1” Boolean variable.

This digital input controls the cooling capacity of EVD_Emb_2 by Sel_INT function block switching between 0 to 100.

Example 3: Analogue positioner

A digital inputneeds to be associated with “DI1” Boolean variable for the command to enable the analogue positioner.

An analogue input is used to get the position as a percentage. It can be set by choosing a 0..10V type probe, setting Min to 0.0 and Max to 100.0 so that it can be connected directly to the input pin ServicePosit_perc_EVDEMB on EVD_Emb_2.
