Appendix 2: differences between Mod_EVDEVO and EVD_Emb_2

Last update 06/05/2015 | Native revision - 20/01/16

 

This section describes the most significant differences between Mod_EVDEVO and the new EVD_Emb_2 function block.

Differences in functions

EVD_Emb_2 manages only one single-pole valve: there are no more references to valve A or B.

EVD_Emb_2 contains the control logic, whereas in Mod_EVDEVO the control logic is contained in the driver valve firmware .

Functions present in Mod_EVDEVO but not implemented in EVD_Emb_2 are listed below:

  • Digital input management (this should be performed externally to the function block);
  • Defrost delay management (this should be performed externally to the function block as it depends on DI);
  • Analogue positioner (this should be performed externally using the manual positioner);
  • Probe alarm management, such as backup probe and emergency valve position (this should be performed externally using the manual positioner).

This has been left to the developer that can implement it as best suit to the application.
