General operations

EnergyMeter_Mono FB is used to acquire voltage, current, power, energy, apparent power and apparent energy data in single-phase systems.

Functions:

  • data acquisition in single-phase systems via Modbus Master protocol
  • acquired value screen display
  • data preparation for remote supervision via BMS
  • data preparation for alarm trigger

Permitted operations:

  • set the device address where data is read
  • enable module operations

See CAREL emeter1 counter (code CAREL MT100D2100).

DevAddr_EMMN selects the device Modbus address where data is acquired.
After acquisition the module displays the acquired values on the terminal after converting them.

Enable the Function Block

In order to correctly enable the function block, the following steps are necessary:

  • instantiate a new EnergyMeter_Mono function block
  • add the device into the modbus protocol line of protocol editor

The following table shows all the parameter shared in the protocol line:

Parameter Description Data types Index
Volt_L1_N Voltage (Volt x 10) Input Register 0
Current_L1 Current in mA Input Register 6
Watt PowerSum (Watt x 10) Input Register 12
KWh_TOT Power Consumption (KWh x 10) Input Register 14
KWh_PAR Partial Power Consumption (KWh x 10) Input Register 17

To enable the module, set En_EMMN = TRUE.

Alarm management

The module detects whether the device, selected by pin DevAddr_EMMN, is connected or not.
If the device is offline, the Al_Offline_EMMN variable is triggered.

Serial settings

Use Modbus Master protocol with the following communication parameters:

Parity none
Baudrate 4800/ 9600
Data lenth 8
Stop bits 2
