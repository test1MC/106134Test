General operations

For the full list of the internal variables available on the board, see chapter 5.1 "Variables accessible via serial" of the device manual

The module is used to configure and manage the KEC controller with the following control modes:

  • Proportional request from the serial line
  • Proportional request from the serial line and flow rate limit
  • Temperature control and flow rate with serial ON/OFF

 

Refer to the user manual +030222165, available on the CAREL website for details about the control modes available.

 

Production is enabled by setting the "OnOffDev_KEC " input to "TRUE" , while the production is managed by setting the "SerialReq_KEC" input, whcih represents the request expressed as a percentage (0.0 - 100.0%) of the rack nominal flow rate. The value of "SerialReq_KEC" is managed only if "OnOffDev_KEC" is "TRUE", otherwise it will be ignored and production will be stopped, even if the request is not zero.

Moreover, the function block manages the device alarm, giving the possibility to monitor device malfunctions, if present, if these do not affect serial communication. If communication between the c.pco controller and the KEC is lost, the KEC controller automatically stops production as soon as no communication is detected.

 

Proportional control from serial line

Production is enabled when "OnOffDev_KEC" is TRUE. Production is set by assigning the proper value (0.0-100.0%, REAL type, percentage of the rack flow rate) to the "SerialReq_KEC" input. Calculation of the production request value is the responsibility of the developer of the application that uses the KEC humidifier.

With those two inputs, the KEC controller will spray the requested amount of water according to request.

 

Proportional control from serial line with water flow limitation

As in the previous case, the function block sends the current production request, calculated by the application, using the values of the "OnOffDev_KEC" and "SerialReq_KEC" inputs. The KEC humidifier will spray the requested amount of water, limiting it in order to optimise water consumption.

 

Temperature control with water flow limitation

When this mode is selected, production is enabled setting the "OnOffDev_KEC" input to TRUE. The unit will operate according to the temperature set point defined by KEC_DEVICE_CTRL.IECTRegSetP using the auxiliary temperature probe connected tochannel U1 on the KEC controller. The production will be calculated in order to optimise water consumption and avoidwaste.
The control algorithm is managed, in this case, directly by the KEC controller, so the application does not have to calculate the request when this control mode is used.

 

KEC is not a stand-alone device. For this reason the function block can't be used only for monitoring purposes (example: the c.pco reads working status). The ON/OFF status (always) and the control request (depending on the settings) need to be sent via serial line in order to permit to the unit to work.

IEC + DEC mode (two zone systems)

The KEC controller can manage two-zone units, one dedicated to Indirect Evaporative Cooling (IEC), which can be managed with the three control modes described above, and one dedicated to Direct Evaporative Cooling (DEC)/Humidification, that can be controlled only with the "Proportional control from serial line" mode.

The "IEC+DEC" mode (two zone) can be set via the KEC_DEVICE_CTRL.ZonesNumber input parameter, where "1" indicates a single zone system (IEC only) and "2" a two zone system (IEC+DEC). It is necessary, together with the configuration described above, to set-up the flow rates of the two zones, while the number of steps is not configurable, because the KEC is able to manage only one step for each zone when used in IEC+DEC mode.

During normal operation, only one zone can be activated, while the other is kept in standby mode. The active zone is determined by the value sent to the "Season_KEC" input. In particular:

  • The IEC rack is activated when "Season_KEC" is TRUE (summer)
  • The DEC rack is activated when "Season_KEC" is FALSE (winter)

The ON/OFF signal and the proportional request are common to the two zones, so the values taken by "OnOffDev_KEC" and "SerialReq_KEC" are applied to the active zone, according to the "Season_KEC" value.

 

Backup unit

The function block includes an embedded backup function used to manage, in an easy and quick way, two different KEC devices, where the second device is simply a backup for the first. A system defined in this way is composed of two devices, a "Main" device and a "Backup" device, which work together as explained below:

  • If no blocking alarm is active, the Main unit will manage production according to user request
  • If a blocking alarm is triggered on the Main unit, the Backup unit is activated
  • If, when the Backup unit is working, no blocking alarms are present on the Main unit (for example, they have been reset), the Backup unit is stopped and the Main unit is activated again.

To manage the backup function simply declare, in the application, two instances of the KEC function block, one for each managed device, and make the below connections:

  • The "BackupReq" input on the Main unit (KEC_1 in the example) must be assigned to "0"
  • The "BackupCall" output on the Main unit (KEC_1) must be connected to the "BackupReq" input on the Backup unit (KEC_2). The "BackupReq" output on the Backup unit can be left unconnected.

The definition of the configuration and request management, which are supposed to be the same on both units, is left to the developer. The integrated backup function does not manage advanced functions such as unit equivalence and rotation. If these features are necessary, we suggest to use other standard or custom management functions that better suit the specific use case.

 

Pumping unit configuration

The KEC function block supports all the pumping units managed by the P+5K0SEA000S0 electronic controller. In order to configure the model, it is necessary to set the KEC_DEVICE_CTRL.CfgModel input to the value that corresponds to the desired unit code, accoridng to the table below.

If the model configured on the P+5K0SEA000S0 does not match the one set in the application, the KEC function block will force the model to the P+5K0SEA000S0. During pumping unit configuration, the reset procedure is started on the P+5K0SEA000S0. During this phase some communication errors may occur. At the end of the procedure it is suggested to check that the communication parameters match both on the master and slave side, if Modbus communication uses a frame different than "19200 8 N 2", which is the defualt on P+5K0SEA000S0.

Alarms

The KEC FB block groups the KEC alarms into two "major" events:

  • Offline alarm, identified by the "Al_Offline_KEC" output, which is triggered when communication between the c.pco and the KEC is lost
  • The KEC alarm, identified by the "AlrmCode_KEC" output that denotes an alarm read from the KEC controller. The event is sent to the application as "alarm code": when more than one alarm is active on the KEC, only the one with the highest priority is noted by the application. The other ones are shown after the one with the highest priority is no longer present.

Support structure

To simplify the creation of the user interface, the function block provides an internal support structure, KEC_MSK_SETTINGS, instanced with the name "MskSettings" that provides the most common enable/disable status to use in the field on the user interface
The developer can use this to easily define the user interface. The function is enabled when the related field is TRUE.
