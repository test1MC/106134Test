How to implement the import/export parameters function in a project - Tutorial

Document version: 1.0

Last update 22/10/2015 | Native revision 22/10/15

 

This tutorial describes how to implement the import/export parameters function from the Demo_ImpExp_Params to a project.

 

Steps
c.strategy
  1. Open Demo_ImpExp_Params
  2. Right click to "Resource" > "Export" > "Export POU's"
  3. Tick the POU "ImpExp" and click "OK"; save the .pp1 file in a directory
  4. Import the .pp1 file in the new project, selecting "Import" or "Import with new name"
  5. POU "ImpExp" and his local variables are now in "Programs"

 

c.mask
  1. Open Demo_ImpExp_Params
  2. Right click to "Layout" >"Export" > "Worksheet as Template"
  3. Tick "03_ImpExp" to export and click "OK"
  4. Select the instance ImpExp to export and save the .wp1 file in a directory
  5. Import the Worksheet as Template in the new project
  6. Select the instance "ImpExp"
  7. Select "Import" or "Import with new name" of worksheet "03_ImpExp"
  8. Select "Import" of all the classes and click "Next"
  9. Select "Do Nothing" of all the ST functions except "ParamImpExp"



  10. Select "Import" of "Arrows.bmp" and click "Finish"
  11. Worksheet "03_ImpExp" is now in "Layout"
  12. Change the IMPEXP loop according to the use of it (it is present a Test mask that can be deleted from the loop)

 

Conclusion

In the project is now present the import/export parameters function that allows to import/export in USB or internal flash memory the DEV variables that have a user value.
