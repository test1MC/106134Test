Demo/Test application for communication in Modbus protocol between c.pCO and pGDTouch:

  • ModBus protocol management on 485
  • ModBus protocol management over TCP/IP
  • Manage of Integer 16/32 bit, real, bool
Project aims

1. Show how a c.PCO can communicate in Modbus with a pGDTocuh.

2. Highlight the different names of the variables between the two different development tools (C.suite and 1toolTouch Editor)

3. Show the different management of the registers to show the “real” variables

 

Project description

The project manages communication between the c.PCO and PGDtouch viaModbus; this Demo includes an example for Modbus 485 and Modbus over TCP/IP

Here the four Modbus data types are managed:“Holding registers, Input Registers, Input Bits and Output Coils”, as well as the followingdata types: “Real, 16-bit integer, 32-bit integer and Boolean"

On the Modbus over TCP/IP page, there is also the possibility to set the c.PCO IP-address using a specific tag type provided by the pGDTouchEditor, called NOIP (Node Override IP); this allows the IP-address that the tags/variables point to, to be modified dynamically and at runtime .

For more information on usingNOIP use, please refer to the 1toolTouchEditor help under the section: Communication protocols -> Node Override IP.

Regarding the “Real” tags/variables, there is different management between the c.Suite and pGDTouchEditor ofthe “endianess” property.

The two systems, in order to correctly communicate the data with each other, inc.Suite modify the “Endianess” property (available under c.design Line-> LineName-> Settings -> advanced), to“Little Endian Swapped (2,1,4,3)”
