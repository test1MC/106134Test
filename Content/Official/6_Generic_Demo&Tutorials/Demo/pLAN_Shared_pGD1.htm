Project description

This demo shows how to share a pGD1 display betweenmultiplec.pCO controllers. By loading the "pLAN_Shared_pGD1.ap1", it can also be used to determine whether a pGD1 bad switching behaviour is related to a hardware or software cause.

It also implements the possibility to let a c.pCO take control of the shared pGD1 display when an event occurs (for example, an alarm), in order to let the pGD1 show the related c.pCO screens.

 

The c.mask function below can be used to define a buttoncombination (in this case UP+DOWN arrows) for switching between the controllers:

The Key switch acts by connectingthe display to the c.pCO with the next pLAN address inthe network.

 

Toallow a c.pCO to take control of the display, the timed loop includes the "AcquireTerminal" function when the "Acquire_term_Event" global variable is equal to 1. One possible solution could be to connecta digital input used as an alarm or another event defined by the user.

Communication settings
Device address Different pLAN address for each c.pCO in the pLAN network
pGD1 configuration Shared configuration for all the c.pCO devices in the network
Serial port pLAN

 

Another possible alternative is to use the c.pCO system screen (pLAN address -> Release Term or Acquire Term)
